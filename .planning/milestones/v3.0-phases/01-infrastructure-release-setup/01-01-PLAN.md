**3. Add theme flash prevention script to index.html (per QA-01):**

The localStorage key is `tt_theme` (verified from useTheme.ts). The app supports three modes: "dark", "light", "system". The script must handle the "system" mode by checking prefers-color-scheme.

Replace the index.html with:
```html
<!doctype html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <link rel="icon" type="image/svg+xml" href="/vite.svg" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>TrustTunnel</title>
    <script>
      // Apply saved theme before React mounts to prevent flash
      (function() {
        try {
          var saved = localStorage.getItem('tt_theme');
          var theme;
          if (saved === 'light') {
            theme = 'light';
          } else if (saved === 'system') {
            theme = window.matchMedia('(prefers-color-scheme: dark)').matches ? 'dark' : 'light';
          } else {
            theme = 'dark'; // default
          }
          document.documentElement.setAttribute('data-theme', theme);
        } catch (e) {
          document.documentElement.setAttribute('data-theme', 'dark');
        }
      })();
    </script>
  </head>
  <body class="bg-surface-950 text-white">
    <div id="root"></div>
    <script type="module" src="/src/main.tsx"></script>
  </body>
</html>
```

Key changes from current index.html:
- Remove `class="dark"` from `<html>` tag (theme is now via data-theme attribute)
- Add inline `<script>` inside `<head>` before `</head>` that reads `tt_theme` from localStorage
- Script handles "system" mode by checking `prefers-color-scheme` media query
- Dark is the default fallback
- IIFE prevents global scope pollution
- try/catch handles localStorage being unavailable (private browsing)
