**Step 1: Install Storybook**

Run from gui-pro/ directory:
```bash
cd gui-pro && npx storybook@latest init --no-install
npm install --save-dev @storybook/addon-themes
npm install
```

Use `--no-install` flag to prevent auto-install (the flag was `--skip-install` in older Storybook versions but renamed to `--no-install` in v10), then run npm install manually to control the process. The init command will auto-detect Vite + React and generate .storybook/main.ts and .storybook/preview.ts scaffolds.

After init, verify that package.json has `storybook` and `build-storybook` scripts added.

**Step 4: Configure .storybook/main.ts**

REPLACE the auto-generated main.ts with:

```typescript
import type { StorybookConfig } from '@storybook/react-vite';
import { fileURLToPath } from 'url';
import path from 'path';

const __dirname = path.dirname(fileURLToPath(import.meta.url));

const config: StorybookConfig = {
  framework: '@storybook/react-vite',
  stories: [
    '../src/**/*.mdx',
    '../src/**/*.stories.@(js|jsx|ts|tsx)',
  ],
  addons: [
    '@storybook/addon-essentials',
    '@storybook/addon-themes',
  ],
  async viteFinal(config) {
    const { mergeConfig } = await import('vite');
    return mergeConfig(config, {
      server: {
        hmr: { host: 'localhost', port: 6007, protocol: 'ws' },
      },
      resolve: {
        alias: {
          '@tauri-apps/api/core': path.resolve(__dirname, './tauri-mocks/api-core.ts'),
          '@tauri-apps/api/event': path.resolve(__dirname, './tauri-mocks/api-event.ts'),
          '@tauri-apps/api/app': path.resolve(__dirname, './tauri-mocks/api-app.ts'),
          '@tauri-apps/api/window': path.resolve(__dirname, './tauri-mocks/api-window.ts'),
          '@tauri-apps/plugin-dialog': path.resolve(__dirname, './tauri-mocks/plugin-dialog.ts'),
          '@tauri-apps/plugin-shell': path.resolve(__dirname, './tauri-mocks/plugin-shell.ts'),
        },
      },
    });
  },
};

export default config;
```

Note: HMR port is 6007 (not 6006, which is the Storybook UI port) to avoid port conflict. The `host: 'localhost'` and `protocol: 'ws'` ensure HMR works in the desktop context. Use `fileURLToPath(import.meta.url)` for cross-platform __dirname in ESM -- do NOT use bare `__dirname` (unavailable in ESM).

**Step 5: Configure .storybook/preview.ts**

REPLACE the auto-generated preview.ts with:

```typescript
import type { Preview, Renderer } from '@storybook/react-vite';
import { withThemeByDataAttribute } from '@storybook/addon-themes';
import '../src/index.css';

const preview: Preview = {
  decorators: [
    withThemeByDataAttribute<Renderer>({
      themes: {
        dark: 'dark',
        light: 'light',
      },
      defaultTheme: 'dark',
      attributeName: 'data-theme',
    }),
  ],
  parameters: {
    backgrounds: { disable: true },
    layout: 'centered',
  },
};

export default preview;
```

The `import '../src/index.css'` loads Tailwind utilities + token CSS into the Storybook iframe. Since main.tsx imports tokens.css then index.css, and index.css contains the @tailwind directives, importing just index.css in preview.ts covers Tailwind. tokens.css is loaded separately by Vite because it's imported in main.tsx -- but Storybook does NOT use main.tsx as entry. So tokens.css needs an explicit import too. Add it:

```typescript
import '../src/shared/styles/tokens.css';
import '../src/index.css';
```

Both imports are needed: tokens.css for CSS custom properties, index.css for @tailwind directives + component utility classes.
