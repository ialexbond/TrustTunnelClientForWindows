# Typography

Font stack: Inter, Segoe UI, system-ui, -apple-system, sans-serif.

## Scale

## Weight 600 (Semibold)

<Typeset
  fontSizes={[11, 12, 14, 16]}
  fontWeight={600}
  sampleText="The quick brown fox jumps over the lazy dog"
  fontFamily='"Inter", "Segoe UI", system-ui, -apple-system, sans-serif'
/>

## Token Reference

| Role | CSS Var | Size | Weight | Line Height | Tracking |
|------|---------|------|--------|-------------|----------|
| xs (caption) | `--font-size-xs` | 11px | 400 | 1.4 | `--tracking-wide` (0.02em) |
| sm (helper) | `--font-size-sm` | 12px | 400 | 1.5 | `--tracking-normal` (0) |
| md (body) | `--font-size-md` | 14px | 400 | 1.5 | `--tracking-normal` (0) |
| lg (heading) | `--font-size-lg` | 16px | 600 | 1.3 | `--tracking-tight` (-0.01em) |

## Design Notes

- Only 2 weights: 400 (normal) and 600 (semibold). No medium or bold.
- Scale steps: 11 -> 12 -> 14 -> 16. Each step is perceptually distinct (min 2px gap).
- 11px captions use wider tracking (0.02em) to prevent character collapse at small size.
- Headings use tighter tracking (-0.01em) for density.

```markdown
# Design Token Architecture

## Overview

Two-tier CSS custom property system in `gui-app/src/shared/styles/tokens.css`.

- **Tier 1 (Primitives):** Raw values in `:root`. Color scales, numeric values. Never referenced directly by components.
- **Tier 2 (Semantics):** Purpose-named aliases in `[data-theme]` blocks. Point to primitives. Components reference only semantics.

Theme switching: `[data-theme="dark"]` and `[data-theme="light"]` redefine semantic tokens. Dark is the default (`:root` inherits dark).

## Accent: Slate-Teal

Desaturated teal/sage. Not indigo (v2 accent, forbidden). Professional on neutral backgrounds.

| Shade | Hex | Role |
|-------|-----|------|
| 400 | #4d9490 | Dark theme interactive (5.87:1 on #0d0d0d) |
| 500 | #2d7a76 | Brand midpoint, decorative only |
| 600 | #236260 | Light theme interactive (6.73:1 on #f9f9f7) |

Semantic aliases: `--color-accent-interactive`, `--color-accent-hover`, `--color-accent-active` -- resolve per theme.

## Color Scales

### Backgrounds
- `--color-bg-primary` -- Window background
- `--color-bg-secondary` -- Sidebar, secondary panels
- `--color-bg-surface` -- Cards (always + `--color-border`)
- `--color-bg-elevated` -- Modals, dropdowns
- `--color-bg-hover` -- Hover state
- `--color-bg-active` -- Active/pressed state

### Text
- `--color-text-primary` -- Body text, headings
- `--color-text-secondary` -- Labels, descriptions
- `--color-text-muted` -- Placeholder, disabled, timestamps
- `--color-text-inverse` -- Text on accent backgrounds

### Status (VPN states)
- `--color-status-connected` -- Green (success-400 dark / success-600 light)
- `--color-status-connecting` -- Amber (warning shade)
- `--color-status-error` -- Red (danger shade)
- `--color-status-disconnected` -- Muted gray
- `--color-status-info` -- Blue (info shade)

Status surfaces: `*-bg` (0.08 opacity), `*-border` (0.15 opacity) for badges/banners.

## Non-Color Tokens

| Category | Tokens | Range |
|----------|--------|-------|
| Spacing | `--space-1` to `--space-8` | 4px to 40px (4px base) |
| Typography | `--font-size-xs` to `--font-size-lg` | 11px to 16px |
| Font weight | `--font-weight-normal`, `--font-weight-semibold` | 400, 600 |
| Letter-spacing | `--tracking-tight`, `--tracking-normal`, `--tracking-wide` | -0.01em to 0.02em |
| Z-index | `--z-base` to `--z-titlebar` | 0 to 500 |
| Shadows | `--shadow-sm` to `--shadow-xl` | Themed dark/light values |
| Focus | `--focus-ring` | Double-ring pattern (bg + accent) |
| Motion | `--transition-fast/normal/slow` | 150ms/200ms/300ms |
| Easing | `--ease-out`, `--ease-in-out` | Cubic bezier curves |
| Opacity | `--opacity-disabled/hover-overlay/pressed-overlay/backdrop` | 0.4/0.08/0.12/0.6 |
| Border | `--border-thin`, `--border-medium` | 1px, 2px |
| Radius | `--radius-sm` to `--radius-full` | 6px to 9999px (preserved from v2) |

## Rules

1. Components reference ONLY semantic tokens (Tier 2), never primitives
2. All `bg-surface` containers MUST use `border: var(--border-thin) solid var(--color-border)` (dark surfaces have minimal contrast)
3. Accent is reserved for 6 interactive element types only (10% rule)
4. No glow effects -- shadows only (restrained elegance)
5. Reduced motion: `@media (prefers-reduced-motion: reduce)` sets all transitions to 0ms

## Files

- Token source: `gui-app/src/shared/styles/tokens.css`
- Deprecated glows: `gui-app/src/shared/ui/colors.ts` (full removal Phase 6)
- Flash prevention: `gui-app/index.html` inline script reads `tt_theme` from localStorage
```

# Phase 1 Decisions

## Accent Color: Slate-Teal

**Decision:** Desaturated teal/sage (#4d9490 dark / #236260 light)
**Why:** Professional, not indigo (v2 forbidden), not blue (common VPN), not generic. Reads as technical/precise.
**Alternatives considered:** Emerald (too nature-y), amber (conflicts with warning), purple (v2-adjacent).
**WCAG:** 5.87:1 dark, 6.73:1 light -- both AA.

## Dark Theme Background: #0d0d0d

**Decision:** Pure neutral near-black, no color undertone
**Why:** Old #0a0a0f had blue undertone (per user D-07). #0d0d0d is perfectly neutral.
**Tried:** #0a0a0a (too indistinguishable from pure black on some monitors). #0d0d0d provides enough lift.

## Light Theme Background: #f9f9f7

**Decision:** Warm cream, not cold white
**Why:** User explicitly wants "warm cream" (D-08). Pure white (#ffffff) feels clinical. Blue-gray (#f5f6fa, old value) had cool undertone.

## Glow Removal

**Decision:** All glow effects set to `none` in colors.ts
**Why:** Restrained elegance direction (D-13). Shadows only. Glow was the most "AI-generated looking" visual element.
**What stayed:** Status dot opacity pulse animation replaces glow.

## Typography: 4-Step Scale (11/12/14/16)

**Decision:** Only 4 font sizes, 2 weights
**Why:** Compact desktop app (not marketing site). More sizes = more inconsistency. 14px body matches Linear/Raycast.

## Spacing: 8-Step Scale (4px base)

**Decision:** --space-1 through --space-8, all multiples of 4
**Why:** 4px base is industry standard (Material, Linear). 8 steps cover all use cases for 32 components. More granularity adds decision fatigue.

## Theme Flash Prevention

**Decision:** Inline script in index.html reads `tt_theme` from localStorage
**Why:** React-based theme application causes white flash on dark-preference loads. Inline script runs synchronously before React mount.
**Key detail:** localStorage key is `tt_theme` (not `data-theme` or `theme`). Supports "system" mode via prefers-color-scheme media query.

## Storybook Tauri Mocks: viteFinal Aliases

**Decision:** Module-level resolve aliases in .storybook/main.ts viteFinal
**Why:** Prevents @tauri-apps/* imports from even loading native modules. Per-story mocking is fragile. viteFinal aliases are global and automatic.
**Mock shapes:** Mirror gui-app/src/test/tauri-mock.ts but use plain ESM exports instead of vi.fn().

## [data-theme] Consolidation

**Decision:** Remove ALL [data-theme] blocks from index.css, keep only in tokens.css
**Why:** DS-02 requirement. Single source of truth for theme-responsive behavior. The 38+ !important overrides in index.css exist because Tailwind utilities can't respond to tokens.
