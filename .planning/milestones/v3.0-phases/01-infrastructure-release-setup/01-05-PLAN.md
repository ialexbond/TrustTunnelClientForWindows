**Part A: Add theme flash prevention to index.html (QA-01)**

Replace the entire gui-pro/index.html with this exact content:

```html
<!doctype html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <link rel="icon" type="image/svg+xml" href="/vite.svg" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>TrustTunnel</title>
    <script>
      // Apply saved theme before React mounts to prevent flash
      (function() {
        try {
          var saved = localStorage.getItem('tt_theme');
          var theme;
          if (saved === 'light') {
            theme = 'light';
          } else if (saved === 'system') {
            theme = window.matchMedia('(prefers-color-scheme: dark)').matches ? 'dark' : 'light';
          } else {
            theme = 'dark'; // default
          }
          document.documentElement.setAttribute('data-theme', theme);
        } catch (e) {
          document.documentElement.setAttribute('data-theme', 'dark');
        }
      })();
    </script>
  </head>
  <body class="bg-surface-950 text-white">
    <div id="root"></div>
    <script type="module" src="/src/main.tsx"></script>
  </body>
</html>
```

Key changes from current:
- Remove `class="dark"` from `<html>` tag -- theme is now via data-theme attribute
- Add inline `<script>` inside `<head>` before `</head>` that reads `tt_theme` from localStorage
- Script handles three modes: "dark" (explicit), "light" (explicit), "system" (checks prefers-color-scheme), default to "dark"
- IIFE prevents global scope pollution
- try/catch handles localStorage being unavailable (private browsing)
- The key `tt_theme` matches exactly what useTheme.ts reads/writes

# Token Architecture -- TrustTunnel v3.0

## Two-Tier System

### Tier 1: Primitives (`:root`)
Theme-independent values. Color scales, spacing, typography, z-index, opacity, border-width, motion, radius.

- **Accent**: slate-teal (#4d9490 dark interactive, #236260 light interactive)
- **Status**: success (green), warning (amber), danger (red), info (blue) -- each with 400/500/600 shades
- **Spacing**: 8 steps, 4px base (--space-1: 4px through --space-8: 40px)
- **Typography**: 4 sizes (xs:11px, sm:12px, md:14px, lg:16px), 2 weights (400, 600), 3 tracking values
- **Z-index**: 6 layers (base:0, dropdown:100, sticky:200, modal:300, snackbar:400, titlebar:500)
- **Shadows**: 4 levels (sm, md, lg, xl) -- theme-dependent opacity
- **Motion**: 3 durations (fast:150ms, normal:200ms, slow:300ms), 2 easings, pulse tokens

### Tier 2: Semantics (`[data-theme="dark"]`, `[data-theme="light"]`)
Theme-dependent values consuming primitives. Backgrounds, text, borders, inputs, toggles, status, shadows, focus ring.

- Dark theme: neutral near-black (#0d0d0d), no blue undertone, accent-400 for interactive
- Light theme: warm cream (#f9f9f7), accent-600 for interactive (higher contrast)

## Accent Reserved For (10% rule)
1. Primary action buttons (connect/apply/save)
2. Toggle "on" state
3. Focus ring (--focus-ring)
4. Active sidebar nav indicator
5. Inline links
6. Storybook accent swatches

## Files
- `gui-pro/src/shared/styles/tokens.css` -- all tokens
- `gui-pro/src/shared/ui/colors.ts` -- legacy inline styles (glow deprecated, removal in Phase 6)
- `gui-pro/index.html` -- theme flash prevention script
