**Input.tsx — full rewrite per D-01, D-05, COMP-04, UI-SPEC:**

Rewrite Input.tsx completely. The component must:
- Use forwardRef wrapping HTMLInputElement
- Use cn() for class merging
- Accept all standard input HTML attributes via spread
- Add new props:
  - `clearable?: boolean` — when true and value is non-empty, show an X (Lucide X icon, 14px) button at right edge of input that clears value via onChange("")
  - `helperText?: string` — renders small text below input in --color-text-muted, --font-size-xs
  - `error?: string` — renders error text below input in --color-status-error, --font-size-xs. When error is present, border becomes --color-danger-500 and background becomes --color-status-error-bg
  - `icon?: ReactNode` — optional left icon slot
- All colors via CSS custom properties only
- No import from colors.ts

Base input styling:
- h-8 w-full rounded-[var(--radius-md)] border border-[var(--color-input-border)] bg-[var(--color-input-bg)] px-[var(--space-3)] text-[var(--font-size-md)] text-[var(--color-text-primary)] placeholder:text-[var(--color-text-muted)]
- focus-visible: border-[var(--color-input-focus)] shadow-[var(--focus-ring)] outline-none
- disabled: opacity-[var(--opacity-disabled)] cursor-not-allowed
- error state: border-[var(--color-danger-500)] bg-[var(--color-status-error-bg)]
- transition: transition-all duration-[var(--transition-fast)] ease-[var(--ease-out)]

Clearable button: absolute positioned right-2, only visible when input has value. Use Lucide X icon, 14px, text-[var(--color-text-muted)] hover:text-[var(--color-text-secondary)]. Wrap input in relative div when clearable or icon is provided.

**Input.stories.tsx — per SB-04, SB-05:**

- argTypes: placeholder (text), helperText (text), error (text), clearable (boolean), disabled (boolean)

**Input.test.tsx — update:**
- Keep existing passing tests, adapt to new API
- Add tests: clearable button appears when value present, clearable calls onChange, helperText renders, error text renders, error applies error classes
