**ProgressBar.tsx — new component per D-11, COMP-12, UI-SPEC:**

Create ProgressBar as named export.

Props:
- `value: number` — 0-100, clamped internally (Math.min(100, Math.max(0, value)))
- `label?: string` — optional label text (rendered as-is per UI-SPEC Copywriting)
- `showValue?: boolean` — when true, shows "XX%" text alongside label
- `className?: string`

Structure:
```
<div className={cn("w-full", className)}>
  {(label || showValue) && (
    <div className="flex justify-between items-center mb-1.5">
      {label && <span className="text-[var(--font-size-sm)] text-[var(--color-text-secondary)]">{label}</span>}
      {showValue && <span className="text-[var(--font-size-xs)] text-[var(--color-text-muted)]">{clampedValue}%</span>}
    </div>
  )}
  <div className="h-2 w-full rounded-[var(--radius-full)] bg-[var(--color-bg-hover)] overflow-hidden" role="progressbar" aria-valuenow={clampedValue} aria-valuemin={0} aria-valuemax={100}>
    <div className="h-full rounded-[var(--radius-full)] bg-[var(--color-accent-interactive)] transition-all duration-[var(--transition-fast)]" style={{ width: `${clampedValue}%` }} />
  </div>
</div>
```

Note: The `style={{ width }}` is acceptable here because width is a dynamic calculated value (not a color). Per COMP-14, colors must be via className tokens, but dynamic widths cannot be done via Tailwind alone.

Accent fill color per UI-SPEC: ProgressBar is one of the 5 components allowed to use accent (--color-accent-interactive).

Track background: --color-bg-hover (subtle, visible track)

ARIA: role="progressbar", aria-valuenow, aria-valuemin=0, aria-valuemax=100

Use cn() for class merging. Zero hardcoded colors.
