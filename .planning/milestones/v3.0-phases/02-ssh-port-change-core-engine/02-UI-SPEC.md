## Spacing Scale

Declared values (source: tokens.css Phase 1 output ‚Äî do NOT invent new steps):

| Token | Value | Usage |
|-------|-------|-------|
| --space-1 | 4px | Icon gaps, inline icon padding, tight compound elements |
| --space-2 | 8px | Compact element internal spacing (badge padding, button gap) |
| --space-3 | 12px | Input internal horizontal padding, list item gaps |
| --space-4 | 16px | Default padding for cards, form sections, modal content |
| --space-5 | 20px | Section header bottom margin |
| --space-6 | 24px | Panel content padding, section gaps |
| --space-7 | 32px | Major vertical rhythm between sections |
| --space-8 | 40px | Top-level layout spacing (reserved for screens, not primitives) |

Exceptions:
- `--space-3: 12px` ‚Äî inherited from Phase 1 tokens.css. This value is NOT on the standard 8-point scale {4, 8, 16, 24, 32, 48, 64}. It IS a multiple of 4 (4√ó3). Used for Input internal horizontal padding and list item gaps. Changing it would break the Phase 1 layout contract. Use only via `var(--space-3)` token reference; do not add new usages.
- `--space-5: 20px` ‚Äî inherited from Phase 1 tokens.css. This value is NOT on the standard 8-point scale {4, 8, 16, 24, 32, 48, 64}. Changing it would break the Phase 1 layout contract. Use only via `var(--space-5)` token reference; do not add new usages.
- `--space-8: 40px` ‚Äî inherited from Phase 1 tokens.css. This value is NOT on the standard 8-point scale. Reserved for top-level layout/screen spacing only ‚Äî Phase 2 primitives must NOT introduce new uses of this value. Changing it would break the Phase 1 layout contract.
- Touch target minimum 32px height (already coded as `h-8` in Button/Input) ‚Äî no change
- Icon-only buttons: 32√ó32px minimum (h-8 w-8)
- Separator: 1px height (horizontal) ‚Äî not a spacing value

## Typography

Source: tokens.css Phase 1 output. All values are locked. Executor must use CSS custom properties, not Tailwind font-size utilities with hardcoded values.

| Role | Size (token) | Weight (token) | Line Height |
|------|-------------|----------------|-------------|
| Caption / badge text | 11px (`--font-size-xs`) | 600 (`--font-weight-semibold`) | 1.2 ‚Äî tight for badges |
| Label / helper text | 12px (`--font-size-sm`) | 400 (`--font-weight-normal`) | 1.4 ‚Äî compact labels |
| Body / input | 14px (`--font-size-md`) | 400 (`--font-weight-normal`) | 1.5 ‚Äî default body |
| Section heading | 16px (`--font-size-lg`) | 600 (`--font-weight-semibold`) | 1.2 ‚Äî headings |

Tracking (letter-spacing):
- `--tracking-tight: -0.01em` ‚Äî section headings
- `--tracking-normal: 0` ‚Äî body, inputs
- `--tracking-wide: 0.02em` ‚Äî badge uppercase text

Rule: exactly 2 weights used throughout Phase 2 ‚Äî 400 and 600. No 500, no 700.

## Color

Source: tokens.css (dark + light semantics). All colors via CSS custom properties ‚Äî zero hardcoded hex in component code.

| Role | Dark value (semantic token) | Light value (semantic token) | Usage |
|------|----------------------------|-----------------------------|-------|
| Dominant (60%) | `--color-bg-primary` (#0d0d0d) | `--color-bg-primary` (#f9f9f7) | App background, modal backdrop base |
| Secondary (30%) | `--color-bg-surface` (#1c1c1c) | `--color-bg-surface` (#f0f0ed) | Cards, form field backgrounds, tooltips |
| Elevated | `--color-bg-elevated` (#222222) | `--color-bg-elevated` (#e8e8e5) | Dropdowns, popovers, floating elements |
| Accent (10%) | `--color-accent-interactive` ‚Üí `--color-accent-400` (#4d9490) | `--color-accent-interactive` ‚Üí `--color-accent-600` (#236260) | Button primary bg, Toggle on-state, Input focus ring, ProgressBar fill |
| Destructive | `--color-destructive` (#e05545) | `--color-destructive` (#b03020) | Button danger variant bg, ErrorBanner error variant border/text |

Accent reserved for (explicit list ‚Äî nothing else uses accent):
1. `Button` ‚Äî `primary` variant background + hover state
2. `Toggle` ‚Äî on-state track fill (`--color-toggle-on`)
3. `Input` + `NumberInput` + `PasswordInput` + `ActionInput` + `ActionPasswordInput` ‚Äî focus border (`--color-input-focus`)
4. `ProgressBar` ‚Äî filled segment
5. Focus ring outline on all interactive elements via `--focus-ring`

Status colors (NOT accent, NOT destructive ‚Äî their own semantic tier):
- Connected: `--color-status-connected` + `--color-status-connected-bg` + `--color-status-connected-border`
- Connecting: `--color-status-connecting` + `--color-status-connecting-bg` + `--color-status-connecting-border`
- Error: `--color-status-error` + `--color-status-error-bg` + `--color-status-error-border`
- Info: `--color-status-info` + `--color-status-info-bg` + `--color-status-info-border`
- Disconnected: `--color-status-disconnected` (= `--color-text-muted`)

Rule: colors.ts references (legacy `colors.*` imports) are removed from every component during Phase 2. Use only CSS custom property vars.

### Accessibility Requirements

**Icon-only buttons (applies to `IconButton` and `Button` with `variant="icon"`):**
- `aria-label` prop is **required** ‚Äî no icon-only button may be rendered without it
- If `aria-label` is absent, the component should log a `console.error` in development (non-breaking)
- TypeScript type: `aria-label: string` must be declared as required in the props interface when rendering in icon-only mode (no visible text child)
- Storybook stories for `IconButton` and `Button variant="icon"` must pass `aria-label` in every story arg

## Interaction States

Every component story must demonstrate these states. Executor implements ALL states, not just default.

| State | Visual specification |
|-------|---------------------|
| Default | Tokens as defined; no outline |
| Hover | `--color-bg-hover` for surfaces; `--color-accent-hover` for accent elements; transition `--transition-fast` |
| Focus-visible | `box-shadow: var(--focus-ring)` ‚Äî 2px gap + 4px accent outline |
| Active / Pressed | `--color-bg-active` for surfaces; scale(0.97) for buttons |
| Disabled | `opacity: var(--opacity-disabled)` (0.4); cursor: not-allowed; no hover/focus effects |
| Error | `--color-danger-500` border + `--color-status-error-bg` background (inputs); `--color-status-error` text (error messages) |
| Loading | Lucide `Loader2` icon, `animate-spin`, replaces button content |

Motion:
- All transitions use `--transition-fast` (150ms) + `--ease-out` for hover/focus
- `--transition-normal` (200ms) + `--ease-in-out` for modal open/close
- `prefers-reduced-motion`: tokens set to 0ms automatically via tokens.css media query

## CVA Architecture

Source: CONTEXT.md D-04, D-05.

Pattern for all CVA components:

```ts
// Naming: camelCase for variant prop keys (matches TypeScript conventions)
// Examples: variant="primary", size="md", severity="error"
const componentVariants = cva(
  "base-classes using-token-vars",  // base
  {
    variants: {
      variant: { primary: "...", danger: "...", ghost: "...", icon: "..." },
      size: { sm: "...", md: "...", lg: "..." },
    },
    defaultVariants: { variant: "primary", size: "md" },
  }
);
```

Rules:
- All colors via `var(--token-name)` ‚Äî zero hardcoded hex in cva strings
- Tailwind utility classes allowed for layout (flex, items-center, rounded-*) when they reference CSS var via bracket syntax
- forwardRef on all interactive components
- `className` prop always merged last (consumer overrides)
- compound variants allowed for complex state combinations (e.g., variant="danger" + disabled)
