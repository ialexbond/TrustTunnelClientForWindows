# Control Panel — Behavior Specification

> Screen: Control Panel (ControlPanelPage.tsx)
> Phase: 3
> Status: Implemented
> Last updated: 2026-04-14

## Overview

Control Panel is the main operational screen. It manages SSH server connection and displays VPN status. It consists of three sub-components: SshConnectForm (first-run login), StatusPanel (VPN state display), and the ControlPanelPage header (server switch).

## Components

| Component | File | Role |
|-----------|------|------|
| ControlPanelPage | `src/components/ControlPanelPage.tsx` | Page orchestrator — routing between SshConnectForm and connected state |
| SshConnectForm | `src/components/server/SshConnectForm.tsx` | SSH credential input form (Card-centered login layout) |
| StatusPanel | `src/components/StatusPanel.tsx` | VPN status strip with StatusBadge + action buttons |

## Dependencies (Design System)

| Component | Primitives Used |
|-----------|----------------|
| SshConnectForm | Card, Button, Input, PasswordInput |
| StatusPanel | StatusBadge, ErrorBanner, Button |
| ControlPanelPage | Button |

## States

### Page States

| State | Condition | Renders |
|-------|-----------|---------|
| Loading | Credentials being read from backend | `null` (no visible flash) |
| No credentials | First run or after server switch | SshConnectForm (centered Card) |
| Credentials loaded | SSH creds exist in backend | Header + StatusPanel + ServerPanel |

### SshConnectForm States

| State | Condition | Visual |
|-------|-----------|--------|
| Empty (initial) | All fields empty | Connect button disabled (opacity-40) |
| Password mode | authMode === "password" | Password field visible, auth toggle: Password active |
| Key-file mode | authMode === "key", keyInputMode === "file" | File picker display + Browse button |
| Key-paste mode | authMode === "key", keyInputMode === "paste" | Textarea for key content |
| Valid idle | Host + credential filled | Connect button enabled (accent fill) |
| Connecting | handleConnect in progress | Connect button: Loader2 spinner + "Подключение..." + disabled |
| SSH error | invoke rejects | SnackBar toast with translated error (auto-dismiss) |
| Host key changed | HOST_KEY_CHANGED error | SnackBar: "Ключ хоста сброшен. Нажмите Подключить снова." |

### StatusPanel VPN States

| VPN State | StatusBadge Variant | Button | Button Label |
|-----------|--------------------|--------------------|--------------|
| connected | connected | danger | Отключить VPN |
| connecting | connecting | ghost (cancel) | Отменить подключение |
| disconnecting | connecting | ghost disabled loading | Отключение |
| recovering | connecting | ghost disabled loading | Восстановление |
| error | error | ghost | Подключить сервер |
| disconnected | disconnected | ghost | Подключить сервер |

### Error Display

| Error Type | Display Method | Behavior |
|------------|---------------|----------|
| SSH connection error | SnackBar (useSnackBar) | Auto-dismiss toast |
| Host key changed | SnackBar (useSnackBar) | Auto-dismiss, key reset silently |
| Persistent VPN error | ErrorBanner severity="error" | User-dismissable, resets on status change |

## Transitions

### No creds -> Connected
1. User fills SshConnectForm fields
2. Clicks "Подключить сервер" -> invoke("check_server_installation")
3. On success: invoke("save_ssh_credentials") -> setCreds() -> page shows Header + ServerPanel

### Connected -> No creds (server switch)
1. User clicks "Сменить сервер" in header
2. invoke("clear_ssh_credentials") -> localStorage cleared -> setCreds(null)
3. Page shows SshConnectForm

### Auto-refresh (wizard install)
1. Wizard sets `trusttunnel_control_refresh` localStorage key
2. ControlPanelPage polls at 500ms interval
3. Detects timestamp change -> re-reads creds from backend -> updates state

## Edge Cases

- **Backend error on credential load**: Returns null -> shows SshConnectForm (graceful fallback)
- **Creds with empty host**: Treated as null -> shows SshConnectForm
- **Creds with no password AND no keyPath**: Treated as null -> shows SshConnectForm
- **Port input**: Strips non-digit characters (e.target.value.replace(/\D/g, ""))
- **Default values**: Port = "22", User = "root"
- **ErrorBanner dismiss**: Local state (errorDismissed), resets when VPN status changes

## Accessibility

| Element | A11y Feature |
|---------|-------------|
| All Buttons | focus-visible ring via --focus-ring token |
| StatusBadge | Readable text content (no aria-label needed) |
| ErrorBanner | role="alert" inherited from component |
| FormField labels | Explicit label prop on Input components |
| Loader2 icons | aria-hidden (visual-only) |
| Button disabled states | disabled attribute prevents interaction |

## i18n Keys Used

### SshConnectForm
control.ssh_title, control.ssh_description, control.connect, control.connecting,
control.auth_method, control.auth_password, control.auth_key, control.key_from_file,
control.key_paste, control.select_key, control.browse, control.remember,
labels.server_ip, labels.port, labels.username, labels.ssh_password,
sshErrors.hostKeyReset

### StatusPanel
status.connected, status.connecting_short, status.disconnecting_short,
status.recovering_short, status.error, status.disconnected,
buttons.disconnect, buttons.connect, buttons.cancel,
vpnErrors.tunnelDown, vpnErrors.authFailed, vpnErrors.generic

### ControlPanelPage
control.disconnect

| Test File | Count | Covers |

| StatusPanel.stories.tsx | All 6 VPN states + AllStates composite + error banner |
| ControlPanelPage.stories.tsx | No-credentials state |
