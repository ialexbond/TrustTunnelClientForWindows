## Spacing Scale

Source: `tokens.css` Tier 1 (4px base, locked in Phase 1)

| Token | Value | Usage in Phase 3 |
|-------|-------|-----------------|
| --space-1 | 4px | Icon-to-label gap, dot indicator gap |
| --space-2 | 8px | Compact field spacing, button icon gap |
| --space-4 | 16px | Card horizontal padding, field-to-field spacing |
| --space-6 | 24px | Header height baseline, between form sections |
| --space-7 | 32px | Form card top/bottom margin from viewport edge |

Exceptions:
- `--space-3` (12px): Phase 1 tokens.css locked design decision — 8-step scale with 4px base includes intermediate values. Used for label-to-input gap and card internal section gap. Overriding this value is not permitted.
- `--space-5` (20px): Phase 1 tokens.css locked design decision — 8-step scale with 4px base includes intermediate values. Used for card vertical padding. Overriding this value is not permitted.
- `--space-8` (40px): Phase 1 tokens.css locked design decision — 8-step scale with 4px base includes intermediate values. Used for vertical centering buffer (SshConnectForm empty area section padding). Overriding this value is not permitted.
- StatusPanel strip height: `52px` (header, 13 × --space-4) — matches existing header rhythm
- StatusPanel border-bottom strip: `52px` (aligns with ControlPanelPage header for visual continuity)
- Textarea (paste key mode): `100px` fixed height — functional constraint, not aesthetic

## Typography

Source: `tokens.css` Tier 1 (locked in Phase 1)

| Role | Token | Size | Weight | Line Height | Usage |
|------|-------|------|--------|-------------|-------|
| Body | --font-size-md | 14px | 400 (--font-weight-normal) | 1.5 | Form labels (via FormField), StatusPanel error text |
| Label | --font-size-sm | 12px | 400 (--font-weight-normal) | 1.4 | FormField labels, secondary metadata |
| Caption | --font-size-xs | 11px | 400 (--font-weight-normal) | 1.3 | Status badge text, uptime counter, security note, auth toggle micro-labels |
| Heading | --font-size-lg | 16px | 600 (--font-weight-semibold) | 1.2 | SshConnectForm card title ("Подключение к серверу") |

Typography rules:
- `--font-size-xs` (11px) and `--font-size-sm` (12px) differ by 1px but serve distinct semantic roles: xs is reserved for captions, timestamps, and uptime counters (subordinate ambient data); sm is reserved for interactive labels, FormField headers, and secondary metadata (content the user reads to act). The distinction is semantic, not purely visual — enforced to prevent role confusion in dense UI areas.
- Letter spacing: `--tracking-tight` (-0.01em) on headings; `--tracking-wide` (+0.02em) on StatusBadge uppercase label
- Monospace: `Geist Mono` on textarea (paste key mode) and SSH key path display
- No arbitrary sizes (no `text-[10px]`, `text-[11px]` Tailwind) — use tokens only

## Color

Source: `tokens.css` Tier 2 semantics (dark + light), Phase 1 D-02, D-07, D-08, D-13

| Role | Dark value | Light value | Usage |
|------|-----------|-------------|-------|
| Dominant (60%) | --color-bg-primary #0d0d0d | --color-bg-primary #f9f9f7 | Page background, ControlPanelPage wrapper |
| Secondary (30%) | --color-bg-surface #1c1c1c | --color-bg-surface #f0f0ed | SshConnectForm Card background |
| Accent (10%) | --color-accent-interactive (400 dark / 600 light) | same | See reserved list below |
| Destructive | --color-destructive #e05545 dark / #b03020 light | same | Disconnect button only |

Accent (`--color-accent-interactive`) reserved for:
1. Primary Button background (SshConnectForm "Подключить сервер" CTA)
2. Selected auth mode button (Password / Key toggle active state)
3. Server icon background in SshConnectForm card header
4. Input focus ring via `--color-input-focus`

Accent NOT used for: general borders, labels, informational text, uptime counter, StatusPanel layout chrome.

Status color usage (semantic tokens, not accent):
- `--color-status-connected` + `--color-status-connected-bg` + `--color-status-connected-border` — StatusBadge `connected` variant
- `--color-status-connecting` + surfaces — StatusBadge `connecting` variant (includes pulse animation on dot)
- `--color-status-error` + surfaces — StatusBadge `error` variant; ErrorBanner `severity="error"`
- `--color-status-disconnected` — StatusBadge `disconnected` variant (transparent bg, muted border)
- `--color-text-muted` — uptime counter, security note, key path helper text

No glow effects. No hardcoded RGBA. No hex outside tokens.css (Phase 1 D-13).

## Component Inventory

All primitives come from Phase 2 output (`gui-pro/src/shared/ui/`).

### SshConnectForm layout

Design choice (03-CONTEXT D-09 — Claude's discretion): **Card centered**, login-screen pattern.
- Outer wrapper: `flex-1 flex items-center justify-center py-[--space-7]` on `--color-bg-primary`
- Card: `w-full max-w-[400px]` using `Card` component (`padding="lg"`, `bg-surface`, `radius-xl`)
- Card header: Server icon (24×24, `--color-accent-interactive`) in 48×48 rounded square (`--color-bg-elevated`, `--radius-lg`), centered; h1 title (`--font-size-lg`, semibold); subtitle caption (`--font-size-xs`, muted)

### SshConnectForm fields

All fields use `FormField` wrapper component:

| Field | Component | Notes |
|-------|-----------|-------|
| Server IP | `Input` (clearable) | Flex-1, placeholder "123.45.67.89" |
| Port | `Input` | w-20 (80px), numeric only, placeholder "22" |
| Username | `Input` | Full width, placeholder "root" |
| Auth mode toggle | `Button` pair (grid-cols-2) | `variant="primary"` for active, `variant="ghost"` for inactive |
| Password | `PasswordInput` | Toggleable visibility |
| Key — file mode | File picker display + `Button variant="ghost"` | Display uses input-chrome styles, filename in Geist Mono xs |
| Key — paste mode | `<textarea>` | 100px height, Geist Mono xs, token-styled (no hardcoded inline style) |
| Key mode toggle | `Button` pair (2 small) | File / Paste sub-toggle under auth=key |
| Connect CTA | `Button variant="primary" fullWidth` | Loading state with `Loader2 animate-spin` |
| Security note | Inline: `Lock` icon + caption | Not a component, just styled text |

Auth mode toggle style: active = `variant="primary"` (accent fill); inactive = `variant="ghost"` (transparent). Remove raw `<button>` elements — use `Button` component everywhere.

### StatusPanel layout

Design choice (03-CONTEXT D-04 — Claude's discretion): **Horizontal strip** (consistent with current UX, refined visually).
- Container: `border-b border-[--color-border]`, height `52px`
- Left: `StatusBadge` (Phase 2) + uptime counter (when connected)
- Right: `Button` (variant depends on VPN state — see state matrix)

VPN state → Button mapping:

| VPN State | StatusBadge variant | Button variant | Button label |
|-----------|--------------------|--------------------|--------------|
| connected | connected | danger | Отключить VPN |
| connecting | connecting | warning | Отмена |
| disconnecting | connecting | warning disabled loading | Отключение... |
| recovering | connecting | warning disabled loading | Восстановление... |
| error | error | primary (ghost) | Подключить сервер |
| disconnected | disconnected | primary (ghost) | Подключить сервер |

StatusBadge dot pulse animation active only on `connecting` variant (already implemented).

Error display in StatusPanel: **remove** `<p style={{ color: "var(--color-danger-400)" }}>` inline error. Replace with `ErrorBanner severity="error" onDismiss={...}` below the strip, inside the StatusPanel container. Short transient errors route to SnackBar (auto-dismiss), persistent VPN errors shown in ErrorBanner (03-CONTEXT D-07, D-08).

### ControlPanelPage Header (when connected)

- Height: `52px` (token: `--space-6 × 2 + button height` — use `h-[52px]`)
- Background: `--color-bg-primary` (via Tailwind `bg-[var(--color-bg-primary)]`)
- Border: `border-b border-[--color-border]`
- Content: `Button variant="ghost" size="sm"` with `LogOut` icon, label "Сменить сервер"
- Alignment: flex end (right-aligned)

Legacy to remove: `style={{ height: 52 }}`, `style={{ backgroundColor: "var(--color-bg-primary)" }}`, `style={{ borderBottom: "1px solid var(--color-border)" }}` — all migrate to Tailwind token classes.

## State Matrix

### Control Panel page states

| State | What renders |
|-------|-------------|
| Loading credentials | `null` (no flash — loading is fast Rust call) |
| No credentials (first run) | `SshConnectForm` (centered card) |
| Credentials loaded | Header (Сменить сервер button) + `StatusPanel` + `ServerPanel` |

### SshConnectForm interaction states

| State | Visual |
|-------|--------|
| Empty (initial) | Connect button disabled (opacity-40) |
| Valid — idle | Connect button enabled (accent fill) |
| Connecting | Connect button: Loader2 + "Подключение..." + disabled |
| SSH error | `useSnackBar()` toast with translated error (auto-dismiss) |
| Host key changed | `useSnackBar()` toast "Ключ сброшен. Нажмите Подключить снова." (auto-dismiss) |

### StatusPanel — uptime counter

Shows only when `status === "connected" && connectedSince !== null`.
Format: `formatUptime(since)` — tabular-nums, Geist Mono, `--font-size-xs`, `--color-text-muted`.
`Clock` icon (14×14) adjacent, `--color-text-muted`.

## Animations and Motion

Source: `tokens.css` motion tokens + `--pulse-duration` (Phase 1)

| Animation | Token | Element |
|-----------|-------|---------|
| Button press scale | `active:scale-[0.97]` | All `Button` components (inherited from Phase 2) |
| State transition | `transition-all duration-[--transition-fast]` 150ms | Button background color changes |
| StatusBadge dot pulse | `animate-pulse` (`--pulse-duration: 2s`) | `connecting` variant dot only |
| Connect button spinner | `animate-spin` | `Loader2` during SSH handshake |
| Error banner entrance | `transition-all --transition-normal` 200ms | ErrorBanner mount (CSS, not JS) |

Reduced motion: `@media (prefers-reduced-motion)` in tokens.css sets all durations to 0ms. No explicit overrides needed in components.

No glow. No blur. No scale transforms beyond active:scale-[0.97].

Destructive actions in this phase:
- "Сменить сервер" (LogOut from ControlPanelPage header) — clears saved credentials, navigates to SshConnectForm. **No confirmation dialog** — user re-enters credentials to reconnect. Decision: 03-CONTEXT D-10 (header redesign includes ghost Button). Rationale: disconnect is low-risk (credentials are re-entered, no data lost).
- "Отключить VPN" (StatusPanel) — stops VPN tunnel. **No confirmation dialog** — immediate action. Reconnect available via "Подключить сервер" button.
- "Отмена" (during connecting) — cancels in-progress SSH handshake. **No confirmation** — obvious cancel.

No modal confirmations required in Phase 3.

## Accessibility Contract

| Requirement | Implementation |
|-------------|---------------|
| Focus ring | `--focus-ring` box-shadow on all interactive elements via `focus-visible:shadow-[var(--focus-ring)] outline-none` (Button, Input, PasswordInput) |
| Status badge | `<span>` with readable text content — no aria-label needed |
| Error messages | `ErrorBanner` renders inline, `role` not required (visible); FormField error uses `role="alert"` |
| Loading state | Button `disabled` attribute set during connecting; `Loader2` is `aria-hidden` (visual only) |
| Icon-only actions | `Button variant="icon"` must have `aria-label` if no text child — not applicable in Phase 3 (all buttons have labels) |
| Input labels | All inputs use `FormField` wrapper with explicit `label` prop |
