# Phase 04 Plan 01: MTProto Proxy Backend + StepProgress + i18n Summary

Rust backend for MTProto proxy management (install/status/uninstall via SSH) with Tauri command registration, StepProgress UI component for step-based progress visualization, and all i18n keys for the Utilities/MTProto section in both Russian and English.

## Decisions Made

1. **mtproto_port naming**: Used `mtproto_port: u16` instead of `port: u16` to avoid collision with the SSH port parameter auto-generated by `ssh_command!` macro.
2. **Co-located tests**: Placed test stubs next to source files (e.g., `StepProgress.test.tsx`) rather than in `__tests__/` directory, matching the existing project convention (15+ test files follow this pattern).
3. **Manual pooled command**: `mtproto_get_status` uses a manual `#[tauri::command]` (like `security_get_status`) instead of `ssh_pool_command!` macro, because it needs to forward the `host` parameter for proxy link construction.
4. **Port validation simplification**: Removed `> 65535` check since `mtproto_port` is `u16` (max 65535), eliminating a compiler warning about useless comparison.
