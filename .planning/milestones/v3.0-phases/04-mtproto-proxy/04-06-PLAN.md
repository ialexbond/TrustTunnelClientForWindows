```markdown
# Application Shell — Use Cases

## UC-SHELL-01: Tab Navigation
**Actor:** User
**Precondition:** App is running
**Flow:**
1. User sees 5 tabs at the top
2. User clicks a tab
3. Content area switches to the selected panel
4. Active tab shows accent underline indicator
**Postcondition:** Selected tab content is displayed

## UC-SHELL-02: Config-Gated Navigation
**Actor:** User
**Precondition:** No VPN config exists (first launch or config cleared)
**Flow:**
1. Only control and about tabs are clickable
2. Other 3 tabs appear dimmed with cursor-not-allowed
3. User sets up connection in control tab
4. After config saved, all 5 tabs become enabled
**Postcondition:** Full navigation available after config setup

## UC-SHELL-03: Keyboard Navigation
**Actor:** User
**Precondition:** App is running, not focused on an input field
**Flow:**
1. User presses Ctrl+1 through Ctrl+5
2. App navigates to the corresponding tab
3. User presses Arrow keys when tab bar is focused
4. Focus moves between tabs
**Postcondition:** Tab changed via keyboard

## UC-SHELL-04: Server Selection (Control Tab)
**Actor:** User
**Precondition:** On control tab with SSH credentials
**Flow:**
1. ServerSidebar shows on the left (200px)
2. Connected server shows green status dot
3. Hover reveals disconnect button
4. Click disconnect stops VPN
**Postcondition:** Server disconnected

## UC-SHELL-05: Window Management
**Actor:** User
**Precondition:** App is running
**Flow:**
1. Title bar shows TrustTunnel PRO with Shield icon
2. User drags title bar to move window
3. User clicks minimize/maximize/close
4. Close button turns red on hover
**Postcondition:** Window moved/resized/closed

## UC-SHELL-06: Update Notification
**Actor:** System
**Precondition:** New version available
**Flow:**
1. Green dot appears on about tab icon
2. User clicks about tab
3. Update info displayed
**Postcondition:** User aware of update
```
