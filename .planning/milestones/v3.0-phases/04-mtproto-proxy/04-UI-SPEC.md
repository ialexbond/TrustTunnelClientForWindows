## Spacing Scale

Source: `tokens.css` Tier 1 (4px base, locked in Phase 1)

| Token | Value | Usage in Phase 4 |
|-------|-------|-----------------|
| --space-1 | 4px | Icon-to-label gap in tab items, status dot margin |
| --space-2 | 8px | WindowControls button internal padding, tab horizontal padding inner |
| --space-3 | 12px | ServerSidebar item horizontal padding, tab active indicator inset |
| --space-4 | 16px | AppName left padding in title bar, ServerSidebar header padding |
| --space-5 | 20px | Tab bar vertical rhythm (height context) |
| --space-6 | 24px | ServerSidebar section gap |
| --space-7 | 32px | Reserved for layout breathing room in shell |

Exceptions:
- Title bar height: **32px** — functional constraint matching current `App.tsx` implementation (Tauri drag region height), kept as-is
- Tab bar height: **40px** — 5 × --space-2 + icon + label fits within this; optically matches the 32px title bar rhythm
- WindowControls button touch target: **40px wide × 32px tall** — matches full title bar height; minimum 40px wide per Windows HIG for close button
- ServerSidebar width: **200px** — matches current implementation, sufficient for server names + status dots; not a spacing token, a layout dimension
- Server list item height: **40px** — matches Sidebar nav item height (36px → rounded up to 40px for easier click targets with host+port two-line display)

Source: `tokens.css` Tier 1 (locked in Phase 1)

| Role | Token | Size | Weight | Line Height | Usage |
|------|-------|------|--------|-------------|-------|
| Body | --font-size-md | 14px | 400 (--font-weight-normal) | 1.5 | Tab labels (active + inactive), ServerSidebar server name |
| Label | --font-size-sm | 12px | 400 (--font-weight-normal) | 1.4 | ServerSidebar secondary line (host:port when label set), ServerSidebar section header |
| Caption | --font-size-xs | 11px | 400 (--font-weight-normal) | 1.3 | "PRO" badge in title bar, WindowControls app name |
| Heading | --font-size-md | 14px | 600 (--font-weight-semibold) | 1.2 | App name "TrustTunnel" in title bar (semibold only, no separate size) |

Typography rules:
- App name in title bar: 14px semibold, `--tracking-tight` (-0.01em), `--color-text-primary`
- "PRO" badge: 11px bold, `px-[--space-2] py-[2px]`, `--color-accent-interactive`, `--color-bg-elevated` fill, `--radius-sm`
- Tab labels: 14px normal weight when inactive, 14px semibold when active — weight shift is the primary active indicator (no color shift alone)
- ServerSidebar section header ("СЕРВЕРЫ"): 11px, `--tracking-wide` (+0.02em), uppercase, `--color-text-muted` — ambient label, not interactive
- No arbitrary Tailwind font sizes. All via token vars

## Color

Source: `tokens.css` Tier 2 semantics (dark + light), 04-CONTEXT D-10/D-11/D-12

| Role | Dark value | Light value | Usage |
|------|-----------|-------------|-------|
| Dominant (60%) | --color-bg-primary #0d0d0d | --color-bg-primary #f9f9f7 | Full window background — title bar, tab bar, content area (seamless, D-10) |
| Secondary (30%) | --color-bg-secondary #161616 | --color-bg-secondary #ffffff | ServerSidebar background — the only column using secondary |
| Elevated | --color-bg-elevated #222222 | --color-bg-elevated #e8e8e5 | Active tab indicator surface, ServerSidebar selected server item |
| Accent (10%) | --color-accent-interactive (400 dark / 600 light) | same | See reserved list below |
| Destructive | --color-destructive #e05545 dark / #b03020 light | same | WindowControls close-button hover only |

Accent (`--color-accent-interactive`) reserved for:
1. Active tab indicator underline (3px line below active tab label)
2. "PRO" badge text color
3. Disconnect button icon (`Power` icon in ServerSidebar server row on hover) — only for connected servers being disconnected

Accent NOT used for: inactive tab labels, title bar text, ServerSidebar borders, general hover backgrounds.

### Seamlessness contract (04-CONTEXT D-09/D-10/D-11)

The shell has **zero hard borders between zones** (title bar / tab bar / content). Separation by surface level only:

| Zone | Background | Separation method |
|------|-----------|-------------------|
| Title bar | --color-bg-primary | None — merges with tab bar |
| Tab bar | --color-bg-primary | None — merges with content |
| Content area | --color-bg-primary | None — full page read |
| ServerSidebar | --color-bg-secondary | Subtle right edge: 1px `--color-border` (rgba 8% white / 9% black) |
| Active tab | --color-bg-primary | 3px accent underline on tab label (not full background fill) |
| Tab hover | --color-bg-hover | Lightweight background on tab pill (no border) |

No `border-bottom` on title bar (D-09 — "seamless like Spotify, Linear").

### Close button hover exception

Windows convention: close button turns red on hover. Use `#e81123` (Windows standard, same as current `WindowControls.tsx`) for hover background. Color is NOT from tokens — it is a platform convention exception. Document in code comment.

```
│                      [Настройки] [О программе]               │
├──────────────────────────────────────────────────────────────┤
│  Content area (flex-1)                                       │
│  ┌────────────────┬─────────────────────────────────────────┐│
│  │ ServerSidebar  │  Panel content                          ││

└──────────────────────────────────────────────────────────────┘
```

### Tab placement relative to WindowControls (04-CONTEXT D-02 — Claude's discretion)

Decision: **Separate rows, title bar above, tab bar below**.

Rationale:
- Windows convention places window controls in the title bar at a specific height (32px) — mixing tabs and controls in one row creates a crowded 5-tab row at full width
- Linear, VS Code, and Figma all separate title bar from nav tabs
- Drag region must span the full title bar width without interference from tab click targets
- Two-row approach gives each zone clear purpose: drag = title bar row; navigate = tab row

Tab bar does NOT carry `data-tauri-drag-region`. The title bar row does.

### ServerSidebar placement (04-CONTEXT D-15 — Claude's discretion)

Decision: **Left column inside content area, visible only on "Панель управления" tab**.

Rationale:
- ServerSidebar is contextual to the Control Panel workflow (select server → connect → manage VPN)
- Showing it on Маршрутизация or Настройки adds noise with no benefit
- Left-column placement is natural — mirrors phase 3 ServerPanel layout, no layout shift on tab switch
- Other tabs render full-width without the sidebar column

Visibility rule:
- `activeTab === "control"` → render ServerSidebar (200px left column) + panel content (flex-1)
- All other tabs → render panel content full-width

### Logo placement (04-CONTEXT — Claude's discretion)

Current sidebar has logo. With sidebar removed, logo moves to title bar left:
- `Shield` icon (16×16, `--color-accent-interactive`) in 28×28 rounded container (`--radius-md`, `--color-bg-elevated`)
- App name "TrustTunnel" (14px semibold)
- "PRO" badge (11px bold, accent-colored)
- Grouped as `flex items-center gap-[--space-2] pl-[--space-4]`
- `data-tauri-drag-region` on the logo group container

### WindowControls redesign

Current state: 3 SVG buttons, `useState` hover, hardcoded `rgba(255,255,255,0.08)` + `#e81123`.

New implementation:
- Button hover background: `--color-bg-hover` via Tailwind token class (not inline style) for minimize/maximize
- Close button hover: `#e81123` bg + `#ffffff` icon (Windows convention, explicit code comment)
- All buttons: `h-full` (full title bar height, 32px), `w-10` (40px)
- No border-radius on window control buttons (matches Windows HIG, no pill shape)
- SVG icons: keep current geometry — minimize dash, maximize square, close ×
- Transition: `transition-colors duration-[--transition-fast]` (150ms) on background

### TabNavigation

| Property | Value |
|----------|-------|
| Height | 40px |
| Layout | `flex items-end` — tabs aligned to bottom of bar (tab indicator sits on the bottom edge) |
| Full width | `w-full` |
| Background | `--color-bg-primary` (seamless, no border-bottom) |
| Gap between tabs | 0 — tabs are flush, no gaps |
| Tab width | `flex-1` (equal width for all 5 tabs on fixed window) |

Single tab anatomy:
```
┌──────────────────┐
│  [icon]  Label   │  ← 40px tall
│          ───     │  ← 3px accent underline (active only)
└──────────────────┘
```

Tab states:
| State | Background | Label color | Weight | Indicator |
|-------|-----------|-------------|--------|-----------|
| Default | transparent | --color-text-muted | 400 normal | none |
| Hover | --color-bg-hover | --color-text-secondary | 400 normal | none |
| Active | transparent | --color-text-primary | 600 semibold | 3px --color-accent-interactive underline |
| Active + hover | --color-bg-hover | --color-text-primary | 600 semibold | 3px underline (unchanged) |
| Disabled | transparent | --color-text-muted opacity-40 | 400 normal | none |

Disabled tabs: tabs requiring config (`hasConfig === false`) render at 40% opacity, cursor not-allowed. On click: do nothing (no error). The "Панель управления" tab is always enabled (shows SshConnectForm if no config).

Active indicator: `position: absolute; bottom: 0; left: 50%; transform: translateX(-50%); width: 60%; height: 3px; background: var(--color-accent-interactive); border-radius: var(--radius-full)` — centered underline, not full tab-width.

Tab icons: lucide-react, 16×16 (`w-4 h-4`), `--color-text-muted` inactive, `--color-text-primary` active.

## Accessibility Contract

| Requirement | Implementation |
|-------------|---------------|
| Focus ring | `--focus-ring` on all tabs via `focus-visible:shadow-[var(--focus-ring)] outline-none` |
| Tab keyboard nav | Arrow keys move between tabs (`role="tablist"`, `role="tab"`, `aria-selected`) |
| Active tab | `aria-selected="true"` on active tab |
| Disabled tab | `aria-disabled="true"`, not `disabled` attribute (still focusable, shows tooltip) |
| WindowControls | `aria-label` on each button (Minimize / Maximize / Close) |
| ServerSidebar disconnect | `aria-label="Отключить {server name}"` on hover-revealed Power button |
| Update badge | `aria-label="Обновление доступно"` on dot indicator span |
| Drag region | `data-tauri-drag-region` attribute on title bar container, NOT on interactive elements |
