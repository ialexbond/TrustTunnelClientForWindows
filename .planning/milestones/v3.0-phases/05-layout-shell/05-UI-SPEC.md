## Spacing Scale

### Наследованная шкала (Inherited Scale)

Все spacing-токены ниже **унаследованы из `gui-pro/src/shared/styles/tokens.css`**, который был установлен до Phase 5 и используется во всём приложении. Phase 5 не вводит новых spacing-значений — она документирует уже существующий контракт кодовой базы.

Три значения (`--space-3: 12px`, `--space-5: 20px`, `--space-8: 40px`) отсутствуют в стандартном наборе GSD {4, 8, 16, 24, 32, 48, 64}, но все они кратны 4px и являются частью **8-ступенчатой шкалы проекта** (space-1 через space-8), объявленной в tokens.css в `:root`. Это не новые дизайн-решения Phase 5 — это унаследованные токены, устоявшиеся в ходе Phase 2–4.

### Объявленные значения

| Token | Value | Usage |
|-------|-------|-------|
| --space-1 | 4px | Icon gaps, внутренние отступы иконок |
| --space-2 | 8px | Compact element spacing, gap между иконкой и лейблом таба |
| --space-3 | 12px | Padding кнопок, gap в sidebar — **унаследован из tokens.css** |
| --space-4 | 16px | Стандартный отступ блоков, px в секциях |
| --space-5 | 20px | Боковой padding контентной области — **унаследован из tokens.css** |
| --space-6 | 24px | Отступ между секциями |
| --space-7 | 32px | Отступ между крупными блоками |
| --space-8 | 40px | Максимальный layout gap — **унаследован из tokens.css** |

**Исключения:**
- TitleBar height: 32px (зафиксировано в Phase 4, не меняется — D-07)
- TabNavigation height: 56px (зафиксировано в Phase 4 — D-04)
- Tab pill size: 120×44px (зафиксировано — D-04)
- ServerSidebar width: var(--sidebar-width-expanded) = 220px (token из tokens.css)
- ServerSidebar header height: 40px (как есть, согласует с общей сеткой)
- Touch target minimum: 44px height (кнопки в sidebar, window controls)
- Tab label font-size: 10px (внутри pill — нарушение 11px minimum, но зафиксировано в Phase 4)

**Источник:** tokens.css `:root` (строки 65-72) — --space-1..8 определены там напрямую; TitleBar.tsx (height: 32), TabNavigation.tsx (height: 56, pill 120x44)

## Typography

| Role | Size | Token | Weight | Weight Token | Line Height | Tracking |
|------|------|-------|--------|--------------|-------------|----------|
| Body | 14px | --font-size-md (text-base) | 400 | --font-weight-normal | 1.5 | 0 (--tracking-normal) |
| Label / UI | 12px | --font-size-sm (text-sm) | 400 | --font-weight-normal | 1.4 | 0 |
| Caption / muted | 11px | --font-size-xs (text-xs) | 400 | --font-weight-normal | 1.3 | +0.02em (--tracking-wide) |
| Heading / active | 14px | --font-size-md (text-base) | 600 | --font-weight-semibold | 1.2 | -0.01em (--tracking-tight) |

**Правила Phase 5:**
- Tab labels: 10px weight-400 (inactive), 10px weight-600 (active) — зафиксировано, не меняется
- Sidebar header: 11px (--font-size-xs) uppercase, weight-400, tracking +0.02em
- Server name в sidebar: 12px (text-sm) weight-500 truncate — единственное исключение для weight-500; для полного соответствия контракту пересмотреть на weight-600
- Только 2 веса: 400 и 600. weight-500 не использовать.

**Унаследованные размеры 10/11/12px:** Три размера в диапазоне 10–12px (1px шаг) унаследованы из Phase 4. Значения 10px (tab labels) и 11px (--font-size-xs) зафиксированы предыдущими фазами и не меняются в Phase 5. Это задокументированное наследие, а не новые типографические решения.

**Источник:** tokens.css (font-size/weight tokens — строки 75-83), memory/v3/design-system.md (правило 2 веса)

## Shell Separation Contract (D-01, D-02, D-03)

Ключевое решение Phase 5 — убрать все `1px solid var(--color-border)` между shell-компонентами.

### Удаляемые бордеры

| Компонент | Текущий border | Замена |
|-----------|----------------|--------|
| TabNavigation | `borderTop: "1px solid var(--color-border)"` | Удалить. Разделение через bg-elevated pill (уже есть) |
| ServerSidebar wrapper | `border-r border-[var(--color-border)]` | Удалить. Заменить на тонкую bg-разницу |
| ServerSidebar header | `border-b border-[var(--color-border)]` | Удалить. Разделение через spacing (pb-1 + pt-2 на списке) |
| ServerSidebar footer (add btn) | `border-t border-[var(--color-border)]` | Удалить. Разделение через py-2 padding |

### ServerSidebar ↔ content разделение (D-03 — Claude's Discretion)

**Решение:** Sidebar получает `--color-bg-secondary` (#161616 dark / #ffffff light) как фон, контентная область остаётся `--color-bg-primary`. Разница 1 уровень иерархии фонов даёт визуальное разделение без бордера. Никакой тени, никакого border.

**Почему:** Паттерн Linear/Notion — seamless design за счёт микро-разницы фонов, не через бордеры. Соответствует философии дизайн-системы (Seamless background principle из memory/v3/design-system.md).

```
Sidebar bg:  --color-bg-secondary (#161616 dark / #ffffff light)
Content bg:  --color-bg-primary   (#0d0d0d dark  / #f9f9f7 light)
```

## Interaction Contract

### Tab Navigation (D-05, D-06)

**Max-width на широких окнах:**
- Tab group (`<nav>`) получает `max-width: 640px; margin: 0 auto; width: 100%`
- Wrapper остаётся `flex items-center shrink-0` на всю ширину
- Это сохраняет равномерное распределение 5 табов на нормальных окнах и центрирует группу на 1200px+
- **Почему max-width: 640px:** 5 tabs × 120px pill + ~8px gaps = ~608px, округляем до 640px

**Roving focus (D-06 — WAI-ARIA tablist):**
- `role="tablist"` уже есть на `<nav>`
- Добавить `onKeyDown` на tablist: `ArrowLeft`/`ArrowRight` перемещают фокус между табами
- `tabIndex={active ? 0 : -1}` на каждом `<button role="tab">`
- Текущий таб получает `tabIndex={0}`, остальные `tabIndex={-1}`
- При переходе стрелками: `element.focus()` на следующий/предыдущий таб (cyclic)
- `Home` → первый таб, `End` → последний таб

### ServerSidebar Show/Hide (D-08, D-09, D-11)

**Условие показа:** `servers.length >= 2`

**Анимация появления/исчезновения (D-11 — Claude's Discretion):**
- CSS transition на wrapper: `width var(--transition-normal) var(--ease-out), opacity var(--transition-normal) var(--ease-out)`
- Состояние hidden: `width: 0; opacity: 0; overflow: hidden`
- Состояние visible: `width: 220px; opacity: 1`
- **Длительность:** `--transition-normal` = 200ms (соответствует Emil philosophy: не анимировать то, что видят 100+ раз/день, но sidebar меняется редко → standard animation)
- `prefers-reduced-motion`: transition мгновенный через media query (уже есть в tokens.css)

**Статус-точки (D-09):** Удалить полностью из ServerSidebar.tsx. Поле `status` в ServerEntry можно оставить в типах (нужно для disconnect кнопки), но визуальный dot — убрать.

### Disconnect button a11y (D-19)

**Текущее:** `opacity-0 group-hover:opacity-100` — невидимо для keyboard-only пользователей

**Исправление:** Добавить `group-focus-within:opacity-100` к классам кнопки disconnect:
```
"opacity-0 group-hover:opacity-100 group-focus-within:opacity-100 ..."
```
Кнопка появляется когда родительский элемент сервера получает фокус через клавиатуру.

### Add Server button (D-10)

**Контракт:** Кнопка "Добавить сервер" НЕ должна вызывать disconnect активного VPN. Должна навигировать к настройке сервера без изменения статуса VPN.

### Tab state caching (D-13, D-14 — Claude's Discretion)

**Решение:** Текущий паттерн `style={{ display: activeTab === X ? "flex" : "none" }}` в App.tsx уже использует CSS display toggle — это НЕ unmount. Проблема с ServerTabs в другом. Диагноз: найти причину ремаунта и исправить без изменения подхода display:none (который уже правильный).

**Проблема:** Кнопки auth (ConnectButton и аналоги) слишком белые на light theme — не хватает контраста.

**Контракт:**

- Active: `--color-accent-active` (accent-800) как bg
- НЕ использовать белый bg с акцентным текстом для primary CTA на light theme

## Design-System Cleanup Contract (D-16, D-17, D-18)

### Sidebar hardcoded colors (D-16)

Все Tailwind arbitrary color classes в ServerSidebar.tsx заменяются на token vars:

| Текущее | Заменить на |
|---------|-------------|
| `bg-[var(--color-bg-elevated)]/50` | `style={{ backgroundColor: "color-mix(in srgb, var(--color-bg-elevated) 50%, transparent)" }}` или убрать opacity-50, использовать `--color-bg-hover` |
| `text-[var(--color-text-secondary)]` | `style={{ color: "var(--color-text-secondary)" }}` |
| `hover:bg-[var(--color-bg-elevated)]` | Через CSS class или style object |

**Правило:** Tailwind arbitrary vars работают для Tailwind-owned properties. Предпочесть `style={{}}` объекты для кастомных CSS переменных (как уже делается в TitleBar/TabNavigation).

### i18n hardcoded strings (D-17)

Компоненты с hardcoded Russian strings:

| Файл | Строки | i18n ключи |
|------|--------|-----------|
| Select.tsx | Любые hardcoded RU строки | `select.*` namespace |
| StatusBadge.tsx | "Подключён", "Подключение...", "Ошибка", "Отключён" | `status.connected`, `status.connecting`, `status.error`, `status.disconnected` |
| EmptyState.tsx | Любые дефолтные RU строки | Передавать через props, не хардкодить |

**Контракт:** Все UI-строки через `t()` из `useTranslation()`. Дефолты в JSON файлах локализации (ru/en).

### sanitize() security fix (D-18)

**Проблема:** sanitize() маскирует только первое вхождение чувствительного ключа в строке. При повторяющихся ключах — утечка.

**Контракт:** Использовать `String.replaceAll()` вместо `String.replace()`, или regex с флагом `g`. Все вхождения должны быть замаскированы.

## Motion Contract

Все анимации согласованы с Emil Kowalski philosophy (skills/emil-design-eng):

| Interaction | Duration | Easing | Properties |
|-------------|----------|--------|------------|
| Sidebar появление | 200ms (--transition-normal) | --ease-out | width, opacity |
| Sidebar исчезновение | 150ms (--transition-fast) | --ease-out | width, opacity |
| Tab hover | 150ms (--transition-fast) | ease | background-color |
| Tab active pill | 150ms (--transition-fast) | --ease-out | background-color, color |
| Disconnect btn fade in | 150ms (--transition-fast) | ease | opacity |
| Window controls hover | 150ms (--transition-fast) | ease | background-color |

**Asymmetric sidebar enter/exit:** появление 200ms (пользователь добавил 2й сервер — важный момент), исчезновение 150ms (быстрый возврат к одному серверу).

**prefers-reduced-motion:** Все transitions обнуляются через media query в tokens.css (уже реализовано).

**НЕ анимировать:** Tab switch — смена таба происходит 100+ раз/день (keyboard shortcut), анимация контента недопустима.
