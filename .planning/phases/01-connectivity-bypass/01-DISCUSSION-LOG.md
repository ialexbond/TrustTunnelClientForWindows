## Fallback-поведение

### Q1: Когда физический адаптер не найден (CONN-04), как себя вести?

| Option | Description | Selected |
|--------|-------------|----------|
| Тихий fallback | Просто использовать дефолтную маршрутизацию без уведомления пользователя | ✓ |
| Лог + fallback | Запись в лог + дефолтная маршрутизация, но без UI-уведомления | |
| UI-уведомление | Показать пользователю, что bypass недоступен и проверки идут через VPN | |

**User's choice:** Тихий fallback
**Notes:** Пользователь не должен думать об адаптерах — это фоновая механика.

### Q2: Если адаптер был, но пропал во время работы, переключаться на дефолт автоматически?

| Option | Description | Selected |
|--------|-------------|----------|
| Авто-переключение | find_physical_adapter_ip() вызывается каждый цикл, при None — дефолт | ✓ |
| Кэширование IP | Найти IP раз при старте, не переопределять | |

**User's choice:** Авто-переключение
**Notes:** Динамическое определение IP каждый цикл мониторинга.
