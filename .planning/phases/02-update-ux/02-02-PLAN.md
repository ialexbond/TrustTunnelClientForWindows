<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| i18n keys → rendered UI | Translation strings are developer-controlled (local JSON files). No user input. |
| updateInfo.releaseNotes → ChangelogModal | Markdown from GitHub API — treated as untrusted content (see Plan 01 threat model). |
| User click → setChangelogOpen | State mutation only. No external side effect. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-05 | Tampering | i18n locale JSON files | accept | Files are developer-controlled, committed to source. No runtime user mutation path. Changes require code deployment. |
| T-02-06 | Information Disclosure | releaseNotes passed as prop from updateInfo | accept | releaseNotes is already fetched from GitHub public API; displayed locally to authenticated desktop user only. No new exposure surface. |
| T-02-07 | Denial of Service | updateInfo.latestVersion undefined → fallback to currentVersion | mitigate | Use `updateInfo.latestVersion ?? version` (where `version` = currentVersion fallback already in component) to avoid rendering "v undefined" in modal header. |
</threat_model>
