## Spacing Scale

Tailwind utilities map to the 8-point scale. The codebase uses Tailwind class names (gap-2, p-4, etc.) with inline `style={}` for token-based colors. Spacing declarations below are the authoritative values for this phase.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps (gap-1), tight inline padding |
| sm | 8px | Button internal padding (gap-2), compact row gaps |
| md | 16px | Card padding (p-4), modal content padding |
| lg | 24px | Modal vertical padding (py-6) |
| xl | 32px | Modal max-height buffer from viewport edge |
| 2xl | 48px | Not used in this phase |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Modal close button touch target: 32px height (h-8), matching existing button pattern
- Modal header padding: 12px vertical (py-3), matching existing card headers

**Source:** Measured from existing `AboutPanel.tsx` card and button patterns.

## Typography

All type sizes follow existing compact UI conventions from `AboutPanel.tsx`. No new sizes are introduced.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 12px (text-xs) | 400 (regular) | 1.5 (leading-relaxed) | Modal markdown body paragraphs, lists |
| Label | 11px (text-[11px]) | 400 (regular) | 1.5 | Modal section labels, muted secondary text |
| Heading-sm | 14px (text-sm) | 600 (semibold) | 1.4 | Markdown `##` headings inside modal |
| Heading-xs | 12px (text-xs) | 600 (semibold) | 1.4 | Markdown `###` headings inside modal |

Markdown `#` (h1) — suppress or render same as heading-sm. GitHub changelogs typically use `##` as top-level version headers.

**Source:** CONTEXT.md D-07 ("text-sm base, text-xs for secondary, headers scaled appropriately"). Matched to existing `AboutPanel.tsx` patterns.

## Component Specifications

### 1. "What's new" Trigger Button

**Location:** Inside the update-available section of `AboutPanel.tsx` and `AboutScreen.tsx`, inline with the existing update row.

**Appearance:**
- Height: h-8 (32px), matching existing action buttons
- Padding: px-3
- Border radius: rounded-lg (var(--radius-lg) = 12px)
- Background: `var(--color-bg-hover)` — neutral, not accent (only visible when releaseNotes non-empty)
- Text color: `var(--color-text-secondary)`
- Font size: text-xs (12px), weight 400
- Icon: `FileText` from lucide-react, w-3.5 h-3.5
- Visible only when `updateInfo.releaseNotes` is non-empty string

**i18n keys (new):**
- `buttons.whats_new` → ru: "Что нового" / en: "What's new"

**States:**
- Default: background `var(--color-bg-hover)`, text `var(--color-text-secondary)`
- Hover: opacity-80 (match existing link pattern) or background `var(--color-bg-active)`
- No disabled state — button only renders when releaseNotes exists

### 2. Changelog Modal

**Structure:**
```
Modal (shared/ui/Modal.tsx — existing component, reuse as-is)
  └── dialog container
        ├── header bar (title + close button)
        ├── scrollable content area
        └── footer bar (close button, secondary)
```

**Container:**
- Width: w-full max-w-md (448px) — fits within 800px app window
- Background: `var(--color-bg-surface)`
- Border: `1px solid var(--color-border)`
- Border radius: rounded-xl (var(--radius-xl) = 16px)
- Shadow: `var(--shadow-lg)` (0 8px 24px rgba(0,0,0,0.5))

**Header bar:**
- Height: 44px (py-3 px-4)
- Background: `var(--color-bg-elevated)`
- Border-bottom: `1px solid var(--color-border)`
- Border-radius top only: rounded-t-xl
- Left: version title — text-sm (14px) weight 600, color `var(--color-text-primary)`
  - Copy: "Что нового в v{latestVersion}" / "What's new in v{latestVersion}"
- Right: X close button — `X` icon from lucide-react, w-4 h-4, color `var(--color-text-muted)`, hover color `var(--color-text-primary)`

**Scrollable content area:**
- Class: `scroll-visible` (defined in index.css — thin visible scrollbar)
- Max height: max-h-80 (320px) — fits within compact Tauri window
- Padding: p-4
- Overflow: overflow-y-auto

**Footer bar:**
- Height: 44px (py-3 px-4)
- Background: `var(--color-bg-elevated)`
- Border-top: `1px solid var(--color-border)`
- Border-radius bottom only: rounded-b-xl
- Contains: single "Закрыть" / "Close" button — h-8, text-xs, background `var(--color-bg-hover)`, color `var(--color-text-primary)`

**i18n keys (new):**
- `modal.changelog_title` → ru: "Что нового в v{{version}}" / en: "What's new in v{{version}}"
- `buttons.close` → ru: "Закрыть" / en: "Close"

### 3. Markdown Content Styling

Rendered by `react-markdown`. Styled via inline `components` prop — no global CSS injection to avoid leaking styles.

| Markdown element | HTML element | Styles |
|-----------------|--------------|--------|
| `# H1` | h1 | Suppress or render as h2 — text-sm font-semibold mt-0 mb-2 color text-primary |
| `## H2` | h2 | text-sm (14px) font-semibold mt-3 mb-1.5 color text-primary |
| `### H3` | h3 | text-xs (12px) font-semibold mt-2 mb-1 color text-secondary |
| Paragraph | p | text-xs leading-relaxed color text-secondary mb-2 |
| `**bold**` | strong | font-semibold color text-primary |
| `*italic*` | em | italic color text-secondary |
| Unordered list | ul | list-disc pl-4 space-y-0.5 mb-2 |
| Ordered list | ol | list-decimal pl-4 space-y-0.5 mb-2 |
| List item | li | text-xs color text-secondary |
| `---` / `***` | hr | border-t border-color-border my-3 |
| Inline code | code | font-mono text-[11px] px-1 py-0.5 rounded bg-color-bg-hover color text-primary |
| Code block | pre > code | Same as inline code but block display, p-3 rounded-lg bg-color-bg-elevated overflow-x-auto |
| Link | a | color accent-500 underline-offset-2 underline opacity-80 hover:opacity-100 — no external navigation |

**Link behavior:** Intercept with `onClick` — call `open()` from `@tauri-apps/plugin-shell` to open in system browser. Do not navigate within the app.

## Interaction Contract

### Open flow
1. User sees update-available row in AboutPanel / AboutScreen
2. "What's new" button visible (only when releaseNotes non-empty)
3. Click opens `<ChangelogModal open={true} ... />`
4. Modal renders with backdrop blur (handled by existing `Modal.tsx`)
5. Content area scrolls independently — app behind modal is inert

### Close flow (three paths — all must work)
1. Click X button in modal header
2. Click backdrop (delegated to existing `Modal.tsx` `closeOnBackdrop={true}`)
3. Press Escape key (delegated to existing `Modal.tsx` `closeOnEscape={true}`)

**Source:** CONTEXT.md "Claude's Discretion" — close behavior.

### Scroll behavior
- Scrollable area uses `.scroll-visible` class (thin 6px scrollbar, matches log panel pattern)
- Scrollbar thumb: `rgba(255,255,255,0.15)` dark / `rgba(0,0,0,0.15)` light (already in index.css)
- Initial scroll position: top (scrollTop = 0 on open)

### Accessibility
- Modal receives focus on open (existing `Modal.tsx` handles portal; add `autoFocus` to close button or container)
- Escape closes modal (existing `Modal.tsx` `closeOnEscape`)
- No ARIA role additions required beyond native `<dialog>`-like semantics (Tauri desktop app, not web)
