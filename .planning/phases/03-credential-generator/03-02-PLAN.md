<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Generator button → controlled input state | onClick sets state directly via React setter — no external call |
| Generated value → form submission | The generated credential is submitted to the VPN server via existing Tauri invoke — not in scope for this phase (already handled) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-04 | Information Disclosure | Password field post-generation | accept | Generated password stays masked (type=password) after generation. User must click Eye to reveal. This is documented UX intent in UI-SPEC. No logging of generated passwords anywhere in the flow. |
| T-03-05 | Tampering | i18n key injection | accept | i18n keys are static strings in bundled JSON files — no runtime eval, no user-supplied input involved. |
| T-03-06 | Denial of Service | Rapid generator button clicking | accept | Each click is a synchronous pure function call with no I/O. No debounce needed; no resource exhaustion risk. |
| T-03-07 | Elevation of Privilege | Generator button when input disabled | mitigate | Add `disabled={isAdding}` prop to generator buttons in both forms (per action section). Disabled button prevents generation during server operations, preventing state inconsistency. |
</threat_model>
