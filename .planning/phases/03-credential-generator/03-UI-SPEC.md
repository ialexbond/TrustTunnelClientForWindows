## Component Inventory

### New component: `GenerateButton`

A small inline icon button placed inside input fields on the right side, using absolute positioning.

```
Pattern source: AddUserForm.tsx lines 46-51 (Eye/EyeOff button)
```

**Structure:**
```
<button type="button" onClick={onGenerate}
  className="absolute top-1/2 -translate-y-1/2 transition-colors hover:opacity-70"
  style={{ color: "var(--color-text-muted)" }}
>
  <Shuffle className="w-3 h-3" />   {/* or Dices — see Icon Selection below */}
</button>
```

**Icon selection:** `Shuffle` from lucide-react.
Rationale: `Shuffle` communicates "random / randomize" clearly.
`Dices` is an acceptable fallback if the developer prefers it — both are already available in lucide-react.

**Icon size:** `w-3 h-3` (12px) — matches Eye/EyeOff icon size in AddUserForm.
In PasswordInput.tsx the Eye is `w-3.5 h-3.5` (14px). Use `w-3.5 h-3.5` when wrapping `PasswordInput` to stay consistent with the eye icon next to it.

### New utility: `credentialGenerator.ts`

Shared utility at `gui-app/src/shared/utils/credentialGenerator.ts`.
Mirrored to `gui-light/src/shared/utils/credentialGenerator.ts`.

No UI surface — pure logic module. No design contract needed beyond the function signatures.

## Icon Placement Spec

### Username field — `AddUserForm.tsx`

Current input: plain `<input>` with `pr-3` right padding, no right-side icons.

After change:
- Add `relative` wrapper div (already present at line 24 in AddUserForm).
- Input gains `pr-8` right padding (matches password field convention).
- Generator button positioned `absolute right-2 top-1/2 -translate-y-1/2`.

```
[ username input text ................... [Shuffle] ]
```

### Password field — `AddUserForm.tsx`

Current input: has Eye toggle at `absolute right-2 top-1/2 -translate-y-1/2`.

After change: two icons on the right, reading left-to-right: `[Shuffle] [Eye]`.

- Input gains `pr-14` right padding (was `pr-8`).
- Generator button: `absolute right-8 top-1/2 -translate-y-1/2` (4px gap from Eye).
- Eye button: stays at `absolute right-2 top-1/2 -translate-y-1/2` (unchanged).

```
[ password input .................. [Shuffle] [Eye] ]
```

### Username field — `UsersSection.tsx`

Current: uses `<Input>` shared component (no right-slot prop exists).
Pattern: wrap `Input` in a `relative` div and overlay the generator button on top,
OR pass an `icon` prop — but `Input.icon` is left-side only.

Resolution: The executor must either:
1. Add an optional `rightIcon` prop to `Input` component, OR
2. Wrap the `<Input>` in a `relative` div and render the button as an overlay sibling.

Recommended: option 2 (no change to `Input` API, lower blast radius).

```
[ username input ................... [Shuffle] ]
```

### Password field — `UsersSection.tsx`

Current: uses `<PasswordInput showIcon={false}>` which already has Eye at `right-2.5`.

After change: generator button at `absolute right-9 top-1/2 -translate-y-1/2` inside a wrapper div.

```
[ password input .................. [Shuffle] [Eye] ]
```

Note: `PasswordInput` Eye is at `right-2.5` (10px). Generator button goes at `right-9` (36px).
Input padding: `pr-16` to accommodate both icons.

## Interaction States

| State | Visual |
|-------|--------|
| Resting | Icon color `var(--color-text-muted)`, no border |
| Hover | `opacity: 70%` transition (150ms, `transition-colors` Tailwind class) |
| Active/click | No special active style — icon fills field immediately |
| Disabled (field disabled) | Generator button also disabled; `opacity: 30%`, `cursor-not-allowed` |
| Post-generation | Field value updates in-place; no animation, no toast |

The generator does NOT show a success indicator. The filled value is the confirmation.
If the password field is in "hidden" mode (type=password), it remains hidden after generation — the user can click Eye to reveal it. This is intentional UX (password stays masked by default).

## Generation Format

Documented here for traceability (implementation detail but affects UX).

**Username:** adjective + "-" + noun, optional 2-digit suffix. Example: `"swift-fox"`, `"bold-eagle42"`.
Charset constraint: `a-zA-Z0-9._-` (enforced by existing `onChange` filter in both forms).
Wordlist size: 20-30 adjective+noun pairs (hardcoded in `credentialGenerator.ts`).

**Password:** 16 random characters. Charset: `a-zA-Z0-9!@#$%^&*`. Uses `crypto.getRandomValues()`.
Existing `onChange` filter in `UsersSection` allows `!@#$%^&*()_+\-=[\]{};':"|,./<>?\`~\\` — generated password charset is a safe subset.
