1. In `gui-pro/src/test/setup.ts`, add a synchronous RAF mock AFTER the existing imports (per D-03, D-11):

```typescript
// Synchronous RAF mock — makes Modal/animation double-RAF execute immediately in tests
global.requestAnimationFrame = (cb: FrameRequestCallback): number => {
  cb(0);
  return 0;
};
global.cancelAnimationFrame = () => {};
```

This makes the double-RAF pattern in Modal.tsx (line 44: `requestAnimationFrame(() => requestAnimationFrame(() => setAnimating(true)))`) execute synchronously, so `mounted` and `animating` both become true immediately in tests.

2. In `gui-pro/src/shared/ui/Section.tsx`, add `aria-hidden` to the collapsible content wrapper (per D-02). Change the collapsible branch (lines 118-126) from:

```tsx
<div
  className="grid transition-[grid-template-rows] duration-200 ease-out"
  style={{ gridTemplateRows: showContent ? "1fr" : "0fr" }}
>
  <div className="overflow-hidden">
    {children}
  </div>
</div>
```

to:

```tsx
<div
  className="grid transition-[grid-template-rows] duration-200 ease-out"
  style={{ gridTemplateRows: showContent ? "1fr" : "0fr" }}
  aria-hidden={!showContent}
>
  <div className="overflow-hidden">
    {children}
  </div>
</div>
```

The `aria-hidden={!showContent}` attribute lets tests use `not.toBeVisible()` for hidden content while keeping the DOM element present for CSS animation.
