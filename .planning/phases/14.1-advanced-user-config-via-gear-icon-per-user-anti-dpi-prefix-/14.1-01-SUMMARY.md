## New Modules

### server_rules.rs
Pure TOML editing (no SSH, no async). Username stored as `# User: {name}` comment via
`toml_edit decor_mut().set_prefix()`. Round-trip preserves comments and formatting.

API: `add_user_rule`, `remove_user_rule`, `find_user_rule`, `UserRule`.

12 unit tests + 2 integration tests (rules_toml_test.rs).

### cert_probe.rs
TLS endpoint probe with `NoopVerifier` — accepts any cert (user is pinning bytes, not validating
CA trust). 10s timeout, 8KB DER size cap (T-14.1-06). Returns `EndpointCertInfo{leaf_der, fingerprint_hex, chain_len}`.

6 unit tests (format_fingerprint, serialization, hostname validation).

### tlv_encoder.rs (Path A)
Base64url decode → TLV stream → append 4 gap TLVs → base64url encode.

| Gap TLV | Tag | Encoding |
|---------|-----|----------|
| skip_verification | 0x07 | bool: [tag, 0x01, 0x01/0x00] |
| certificate | 0x08 | bytes: [tag, varint(len), der...] |
| upstream_protocol | 0x09 | varint: [tag, varint(vlen), varint(value)] |
| anti_dpi | 0x0A | bool: [tag, 0x01, 0x01/0x00] |

Omission rules match upstream: bool false → omit; "auto"/None → omit; None bytes → omit.
11 unit tests covering all 4 gap TLVs, error cases, and varint encoding.

### Approach Changes

**Prefix generation — client-side random instead of CLI dry-run:**
Plan suggested attempting `--dry-run` CLI invocation to generate prefix. Abandoned:
1. Endpoint CLI `--dry-run` flag existence not confirmed
2. Client-side `rand::thread_rng()` is deterministic, testable, and secure
3. Simpler code path — no extra SSH round-trip

Result: generated prefix is 4-byte (8 hex chars) random value. Matches the pattern used by upstream prefix-length=4 default.
