# Plan 14.1-02 — CIDRPicker primitive

Создан новый UI-primitive `CIDRPicker` для структурированного ввода IP+prefix, консьюмится UserModal Deeplink-секцией (D-8).

## Task 1 — `cidr.ts` pure utilities + 27 tests (commit 0170619d)

**Выдано:**
- `isValidCidr(s)` — whitelist `[0-9./]`, октеты 0..=255, prefix 0..=32, пустая строка ≡ "нет ограничений"
- `parseCidr(s)` — возвращает `{ octets, prefix } | null`
- `formatCidr(octets, prefix)` — empty при любом пустом октете (partial state = empty semantic)
- `describeCidr(s)` — i18n-ключи для пустого/0.0.0.0/0 + вычисляемый range для валидных CIDR

27 assertions: 11 isValidCidr (включая shell-injection rejection) + 4 parseCidr + 7 formatCidr + 4 describeCidr + 1 bug fix.

## Task 2 — `CIDRPicker.tsx` component + 13 tests (commit bd6653ee)

**Выдано:**
- React-компонент с props: `value, onChange, onError?, disabled?, label?, helperText?, className?, aria-label?`
- Композиция 4×`NumberInput` (min=0, max=255, w-14) + dot/slash separators + `Select` (33 prefix options)
- `PREFIX_OPTIONS` const exported для tests (W12-ревизия)
- useMemo-обёртка на `octets` array → стабильные useCallback deps
- Auto helper-text через `describeCidr()` при отсутствии prop `helperText`
- Empty-state (любой октет пуст) → `onChange("")` = нет CIDR-ограничения

**Barrel export** (`shared/ui/index.ts`): `CIDRPicker, PREFIX_OPTIONS, CIDRPickerProps` добавлены алфавитно между `Card` и `ConfirmDialog`.

13 test cases: render + parsing + onChange + clearing + PREFIX_OPTIONS.length=33 + disabled + aria-label + helperText + label + onError.
