### CR-01: ABI mismatch — `pin_certificate_der` / `leaf_der` now base64 string

**Files modified:**
- `gui-app/src-tauri/src/ssh/server/cert_probe.rs`
- `gui-app/src-tauri/src/ssh/server/mod.rs`

- `gui-app/src-tauri/src/ssh/server/server_config.rs`
- `gui-app/src-tauri/src/commands/ssh_commands.rs`

**Commit:** `86c3a070`

**Applied fix:**
- `EndpointCertInfo.leaf_der: Vec<u8>` renamed to `leaf_der_b64: String` (base64 of DER).
- New `decode_cert_der_b64(&str) -> Result<Vec<u8>, String>` centralises base64 decoding plus the 8 KB size cap (`MAX_CERT_DER_BYTES` promoted to `pub const`).
- `server_add_user_advanced`, `export_config_deeplink_advanced`, and the two Tauri pool command wrappers now accept `pin_certificate_der: Option<String>` (base64). Raw bytes are produced once at the boundary and flow into `tlv_encoder` unchanged.
- `server_add_user_advanced` re-encodes bytes back to base64 before calling `export_config_deeplink_advanced` to preserve the public wire format.
- Added 3 tests: `decode_cert_der_b64_roundtrips`, `decode_cert_der_b64_rejects_invalid_base64`, `decode_cert_der_b64_enforces_size_cap`.
- `endpoint_cert_info_serializes` updated — asserts `"leaf_der_b64":"<base64>"` string in the JSON output.
- Frontend (`CertificateFingerprintCard.tsx`) already expected the new shape; the backend now produces it, closing the loop.

### CR-03: Shell injection in password rotation

**Files modified:**
- `gui-app/src-tauri/src/ssh/sanitize.rs`
- `gui-app/src-tauri/src/ssh/server/server_install.rs`
- `gui-app/src/shared/i18n/locales/ru.json`
- `gui-app/src/shared/i18n/locales/en.json`
- `gui-app/src/components/server/UserModal.tsx`
- `gui-app/src/components/server/PasswordRotationPrompt.tsx`

**Commit:** `5900d4d5`

**Applied fix (combo strategy):**
1. `server_rotate_user_password` switched from `python3 -c "..."` (double-quoted bash) to a single-quoted heredoc `python3 << 'PY_ROTATE_EOF'`. Bash performs no substitutions inside the heredoc body — the only terminator is a literal delimiter line — so `$`, `` ` ``, `\`, `"` can no longer escape to shell context.
2. `validate_vpn_password` extended to reject `'`, `"`, `\` (breaks python literal or TOML value boundaries). `$`, `` ` ``, `#` stay allowed (safe inside single-quoted heredoc body and the TOML `[^"]*` regex).
3. Frontend mirrors the backend validator via new i18n key `server.users.password_no_quotes_backslash` (RU/EN) — surfaced in both `UserModal` and `PasswordRotationPrompt`.
4. Tests: 4 new password cases covering `\`, `'`, `"`, the `evil"; rm -rf /; #` payload from the review, plus positive cases for `$`, `` ` ``.

> Logic note: the heredoc rewrite is syntactic, not semantic — reviewing the actual rotation flow against a real Linux endpoint is prudent. Flagged for human verification in phase exit review.
