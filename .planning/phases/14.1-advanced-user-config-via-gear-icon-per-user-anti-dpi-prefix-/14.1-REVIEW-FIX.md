## Fixed Issues

### WR-03: `server_regenerate_client_prefix` silently drops existing CIDR (security regression)

**Files modified:** `gui-pro/src-tauri/src/ssh/server/server_install.rs`
**Commit:** `49dae9bb`
**Applied fix:**
Before delegating to `server_update_user_config`, read `rules.toml` via
`exec_command` + parse with `super::find_user_rule` to recover the
existing `cidr` for the target user. Pass that through as the `cidr`
argument so the subnet restriction survives the prefix rotation instead
of being silently dropped. Read failure falls back to `None` (pre-fix
behaviour), so a transient SSH error surfaces the same way as before.

Comment block documents the WR-03 security rationale so the invariant
is visible to future readers.

### WR-01: Fire-and-forget Promise pattern hides real SSH write failures

**Files modified:**
- `gui-pro/src/components/server/UsersSection.tsx`
- `gui-pro/src/components/server/UserModal.tsx`
**Commit:** `0c57b4c6`
**Applied fix:**
The `Promise.resolve(invoke(...)).catch(...)` wrappers around
`server_set_user_advanced` / `server_delete_user_advanced` calls were a
defensive pattern for Vitest mocks that return `undefined` (bare `.catch`
on a non-Promise throws sync). The wrapper worked but obscured the intent.

Replaced with async IIFE — `void (async () => { try { await invoke(...); }
catch (err) { activityLog(...) } })()`. Benefits:
- `await invoke(...)` handles both real Promises and mock-undefined;
- `try/catch` structurally documents intent;
- SSH write failures now reach activity.log via an explicit ERROR line.

No UI behaviour change — success path fires immediately as before, only
true backend failures get logged now.

Test suite: 84/84 passed across UsersSection, UserModal, CIDRPicker.

### WR-02: Stale closure risk in `UsersSection` tab-activation effect vs SSH-port-change flow

**Files modified:** `gui-pro/src/components/server/UsersSection.tsx`
**Commit:** `c4d632e8`
**Applied fix:**
Applied the `OverviewSection.rebootRefs` pattern (lines 262–312) to the
M-04 tab-activation effect. A `tabRefs` `useRef` is updated on every
render with the current `{ sshParams, loadServerInfo, refreshDisplayNames,
activityLog }`. The effect reads `tabRefs.current` so each activation uses
fresh values without taking them as explicit deps — which would force the
effect to re-fire every time the SSH pool rebuilt its params object.

Removed the now-redundant `eslint-disable-next-line react-hooks/
exhaustive-deps` directive — the refs.current read makes the minimal deps
array intentional, not a smell.

### WR-04: systemctl restart blast radius — operator UX leaves active clients to be disconnected silently

**Files modified:** `gui-pro/src-tauri/src/ssh/server/server_install.rs`
**Commit:** `bd29f40b`
**Applied fix (partial — coalesce applied, UI banner deferred):**
Backend coalescing:
- Split `add_server_user` into public wrapper (restart=true, used by the
  standalone Tauri command) and internal `add_server_user_internal`
  (parameterised with `skip_restart: bool`).
- `server_add_user_advanced` now calls the internal variant with
  `skip_restart=true` and emits a single coalesced restart at the end of
  Step 2, so both `credentials.toml` and `rules.toml` writes activate in
  one restart cycle.

Blast radius: other clients now see **one** ~2-5s dip per admin action
instead of two. No behavioural change for the non-advanced Add path —
`skip_restart` defaults to `false` in the public wrapper.

Follow-up (deliberately not in this commit):
- UI info-banner in `UserModal` confirming that saves briefly disconnect
  other clients — deferred to a future UX phase so i18n parity + layout
  decisions are made together with the rest of Activity-Log UX.
- Investigate endpoint CLI `SIGHUP` / `systemctl reload` so rules-only
  changes can avoid restart entirely (infra task, out of scope here).

### WR-05: CIDR paste handler under-validates octet count + silent drops beyond 4

**Files modified:**
- `gui-pro/src/shared/ui/CIDRPicker.tsx`
- `gui-pro/src/shared/ui/CIDRPicker.test.tsx`
**Commit:** `419ba68b`
**Applied fix:**
Two silent-truncation edge cases closed:
1. `"10.20.30.40.50"` (5 groups) → added `if (nums.length > 4) return false;`
   Ambiguous paste falls through to default single-field paste, so the
   bogus value lands visibly in one octet instead of silently producing
   a "valid" 4-octet IP.
2. `"10.0.0.0/3a4"` (malformed prefix) → tightened prefix parse to
   `/^\d{1,2}$/` after `trim()`. Non-digit characters anywhere in the
   prefix segment → prefix stays at existing value; no silent "partial
   extraction" any more.

Added 5 unit tests in `CIDRPicker.test.tsx`:
- rejects >4 numeric groups
- rejects malformed prefix (`/3a4`)
- rejects out-of-range `/99` (2-digit but > 32)
- accepts 2-digit boundary `/32`
- accepts prefix with surrounding whitespace

All 23 tests pass (18 prior + 5 new).
