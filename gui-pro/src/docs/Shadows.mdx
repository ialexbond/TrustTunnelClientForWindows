# Shadows

Shadow-система TrustTunnel — **5 токенов для elevation + focus ring**. No glow, no color-tint aura. Restrained elegance.

**Философия:** тень обозначает **высоту** элемента над surface (elevation). Чем выше — тем шире размытие, тем больше смещение. Никаких colored glow effects (accent-tint / success-tint / danger-tint) — они визуально шумят и противоречат минимализму.

**Two tiers per theme:** dark theme использует более сильную alpha (0.20–0.64) чтобы shadow читался на тёмном фоне; light theme — нежнее (0.04–0.16).

## Focus ring

Отдельная система. Используется на ВСЕХ interactive elements через `:focus-visible`.

```css
--focus-ring: 0 0 0 2px var(--color-bg-primary), 0 0 0 4px var(--color-accent-interactive);
```

Double-ring pattern — внутреннее 2px в цвет фона (создаёт зазор), внешнее 4px accent (видно на любой surface).

**Usage:** `focus-visible:shadow-[var(--focus-ring)]` в Tailwind class. Применён в Button, IconButton, ActionInput, CIDRPicker, Toggle, Accordion, и т.д.

**Не путать с `--shadow-*`** — focus-ring это outline/ring effect, не elevation.

## Rules

### Elevation system
- **5 levels only** — `xs` / `sm` / `md` / `lg` / `xl`. Не ad-hoc shadow values.
- **Elevation = tier, not blur.** `--shadow-md` не "medium blur", а "raised above flat surface".
- **Theme-aware opacity** — dark theme 4x stronger alpha чем light. Управляется tokens, не overridden.

### Tailwind usage
- **Bracket notation required** — `shadow-[var(--shadow-sm)]` (НЕ `shadow-sm` Tailwind native — default values не совпадают с токенами).
- **Inline style OK** для single-use contexts — `style={{ boxShadow: "var(--shadow-lg)" }}`.

### Focus / ring
- **Focus states** — always `--focus-ring` через `focus-visible:shadow-[var(--focus-ring)]`.
- **Non-focus ring accents** (active step, selection) — solid color `0 0 0 2px var(--color-accent-500)`, НЕ tint colors.

---

## Don'ts

- ❌ **Glow effects** — `boxShadow: "0 8px 32px var(--color-accent-tint-30)"` или подобное. Tint-based shadow = glow. **Запрещено в production UI.** Используй `--shadow-*` (elevation) или none.
- ❌ **Arbitrary pixel values** — `boxShadow: "0 10px 30px rgba(0,0,0,0.2)"` inline. Используй token.
- ❌ **2-layer glow** — `boxShadow: "0 8px 32px tint1, 0 0 64px tint2"` (WelcomeStep old). Всегда один `--shadow-*` token.
- ❌ **Tailwind native** `shadow-sm/md/lg/xl/2xl` без bracket — они используют Tailwind defaults, не наши токены.
- ❌ **Inline `box-shadow: 0 ...` в CSS** — используй `var(--shadow-N)`.
- ❌ **Color-tinted focus ring accent glow** — `0 0 0 2px var(--color-accent-tint-50)` создаёт soft-glow edge. Используй solid `var(--color-accent-500)` или `--focus-ring`.

---

## Exceptions (задокументированные отклонения)

Эти места legitimate используют custom shadow — **НЕ тиражировать**, но есть rationale.

| Element | Value | File | Rationale |
|---|---|---|---|
| `.wizard-input:focus` | `0 0 0 2px var(--color-accent-tint-15)` | index.css | Subtle tinted focus для form inputs — между default (none) и full focus-ring. Pre-existing pattern. |
| StepBar active indicator | `0 0 0 2px var(--color-accent-500)` | StepBar.tsx | Solid-accent ring для active step — НЕ glow (нет blur, solid color), visually отличается от focus-ring (simpler). |
| `boxShadow: "none"` explicit | — | ServerStatusSection.tsx | Intentional reset (parent's shadow shouldn't cascade). |
