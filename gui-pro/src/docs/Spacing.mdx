# Spacing

Spacing-система TrustTunnel (Pro & Light). **8 токенов на 4px сетке.**

**Философия:** все margin / padding / gap — кратны 4 пикселям. Меньший шаг размывает, больший — ломает ритм. 4px — industry standard (Material Design, Apple HIG, Tailwind).

---

## Decision tree

### «Gap между иконкой и текстом в строке?»
→ `gap-2` (8px / `--space-2`) — industry-standard compact inline spacing.

### «Input internal padding (vertical)?»
→ `py-[var(--space-3)]` (12px) — ⭐ dominant pattern в codebase.

### «Gap между form fields?»
→ `gap-4` (16px / `--space-4`) — default, applied в FormField / ConfirmDialog / UserModal.

### «Card padding?»
→ `p-4` (16px) compact · `p-[var(--space-5)]` (20px) default · `p-6` (24px) generous.

### «Modal padding?»
→ `p-[var(--space-6)]` (24px) — heavier than card, gives breathing room to dialogs.

### «Section break (major break между logical зонами)?»
→ `mt-[var(--space-7)]` (32px) или больше.

### «Centered empty/error state height?»
→ `py-[var(--space-7)]` (32px, EmptyState pattern) или больше.

### «Icon gap в tight контексте (badge с dot-индикатором)?»
→ `gap-1.5` (6px, Badge base) — sub-token, only для compact pill UI.

## Rules

### Base grid
- **4px unit** — весь spacing кратен 4 пикселям.
- **Min:** `--space-1` (4px) — меньше нет смысла.
- **Max:** `--space-8` (40px) — больше требует дизайн-решения, не ad-hoc.

### Tailwind ↔ tokens mapping

- **`p-1..p-6`, `m-1..m-6`, `gap-1..gap-6`** — матчат `--space-1..6` напрямую (4/8/12/16/20/24px). **Используй Tailwind classes — проще и читаемее.**
- **`p-7..p-10`** — Tailwind default values (28/32/36/40) **НЕ совпадают** с нашими `--space-7..8`. **Не используй.**
- **Для 32/40** — всегда `p-[var(--space-7)]` / `p-[var(--space-8)]` (bracket notation с токеном).

### Bracket notation

- **`p-[var(--space-N)]`** — всегда безопасно, читается как "token-based spacing".
- **`p-[Npx]`** с произвольным N — **только если N = multiple of 4**. Иначе anti-pattern.

---

## Don'ts

- ❌ **`padding: '13px'` / `padding: '15px'`** (inline или через `p-[13px]`) — не multiple of 4, ломает ритм.
- ❌ **`p-7` / `p-8` / `p-10`** (Tailwind native high indices) — 28/32/40px, **не совпадают** с `--space-7..8` (32/40px). Naming-collision guaranteed. Используй bracket.
- ❌ **Inline `style={{ padding: '16px' }}`** — используй Tailwind class `p-4` или `p-[var(--space-4)]`.
- ❌ **Magic numbers в CSS** — если в `.css` / `.module.css` пишешь padding/margin, всегда `var(--space-N)`, не сырые px.
- ❌ **Смешение конвенций** — в одном файле или `p-4` (Tailwind), или `p-[var(--space-4)]` (bracket). Не миксуй в одной строке.
- ❌ **`space-1/2/3...` как Tailwind class без префикса** — `space-x-*`/`space-y-*` в Tailwind это child-item spacing (margin-between), не «наш spacing token». Не путать.

## Exceptions (non-4-grid, задокументировано)

Эти места осознанно отклоняются от 4px grid — не anti-pattern, но **не тиражировать**.

| Element | Value | File | Rationale |
|---|---|---|---|
| Sidebar collapsed width | 56px | `--sidebar-width-collapsed` | Icon-only nav column, fits 32px icon + padding |
| Sidebar expanded width | 220px | `--sidebar-width-expanded` | Standard sidebar width, not spacing token |
| Window control button | 30×26px | `index.css:.window-control-btn` | OS chrome, Windows 11 DWM conventions |
| TitleBar PRO badge padding | `pt-[3px]/pb-[2px]` | `TitleBar.tsx` | Optical vertical alignment в 32px bar (true center + offset for cap-height) |
| AboutPanel PRO badge | `pt-[4px]/pb-[3px]` | `AboutPanel.tsx` | Optical vertical alignment в compact badge |
| Touch / click target min | 32px | `--space-7` | Accessibility minimum (32px matches `h-8`) |

## Cyrillic / i18n

Spacing независим от языка. Но **длинные слова** в узких контейнерах (cards 140px wide) могут требовать больше horizontal padding для читаемости. Если RU-label «Переподключиться» не влезает — увеличь `px-*` на одну ступень, не уменьшай `text-*`.
