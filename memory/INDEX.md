# TrustTunnel — рабочая память проекта

`memory/` — это внутренняя рабочая база знаний по TrustTunnel Client.

Это не пользовательская документация и не маркетинговые тексты. Здесь лежат:

- технические reference-заметки
- архитектурные и продуктовые решения
- журналы сессий
- changelog значимых изменений
- v3 UI/UX-материалы
- локальный backlog

## Как читать memory

- Текущая реализация desktop UI живет в `gui-pro/` и `gui-light/`.
- `.planning/` хранит milestone-статус, фазовые планы и follow-up задачи.
- `memory/` хранит объяснения, контекст, решения и reference.

Если документы расходятся между собой, приоритет такой:

1. текущий код в `gui-pro/` и `gui-light/`
2. самые свежие заметки в `.planning/`
3. старые `memory/` reference-документы

Важно:

- в части старых заметок еще встречается путь `gui-app/`
- для текущего desktop v3 это уже исторический след, а не основной source of truth
- `memory/reference/reference/` — старый дублированный mirror reference-файлов; использовать его как основной источник не нужно

## Основные разделы

- `reference/` — актуальные технические срезы по архитектуре, frontend, backend, networking, installer и workflow
- `decisions/` — архитектурные и продуктовые решения с контекстом и мотивацией
- `v3/` — материалы по редизайну, дизайн-системе и экранным контрактам
- `sessions/` — журналы рабочих сессий
- `changelog/` — подтвержденные значимые изменения
- `todo/` — backlog, follow-up идеи и незавершенные хвосты
- `_templates/` — шаблоны для новых заметок

## Что читать в начале задачи

- `reference/architecture.md` — общая карта проекта
- `reference/frontend.md` — текущее состояние `gui-pro` и `gui-light`
- `reference/rust-backend.md` — Tauri/Rust слой
- `decisions/v3-philosophy.md` — философия контрактной разработки
- `decisions/v3-design-guidelines.md` — визуальное направление v3
