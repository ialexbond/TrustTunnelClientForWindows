# v3.0 — Дизайн-ориентиры и скоуп

**Дата:** 2026-04-13
**Контекст:** Exploration-сессия, определение визуального направления v3.0

## Цель редизайна

Перейти от «выглядит неплохо, но видно вайб-кодинг» к «видно, что здесь был план, видна современность приложения». На дворе 2026 год — приложение должно выглядеть качественно, стильно и не похоже на типичный AI-сгенерированный проект.

## Визуальные ориентиры

### Уровень качества
- **Apple Design System** и **Google Material Design** — как эталон ощущения: чисто, системно, дорого-богато
- Не копировать, а достичь того же уровня проработки и внимания к деталям

### Функциональный референс
- **Amnezia VPN** — ближайший аналог по функциональности (для Lite-версии)
- Для Pro-версии прямого референса нет — ближайший по маршрутизации: NekoBox

### Антипаттерн
- «AI-лицо» — одинаковые подходы к компонентам, отступам, цветам, которые выдают вайб-кодинг
- Банальные, простые цвета без индивидуальности
- Отсутствие единой системы → каждый экран как отдельный проект

## Дизайн-требования

### Общие
- Свежий, современный вид (2026 год)
- Новые цвета, особенно тёмная тема
- Новые акцентные цвета, гармонично сочетающиеся между собой
- Единая дизайн-система для всего приложения
- Кастомное окно Windows (не стандартный title bar)
- Приложение не должно быть похоже на типичный AI-сгенерированный проект

### Pro и Lite — одна семья
- Общая дизайн-система (токены, компоненты, типографика)
- Индивидуальность каждой версии (разные акценты? разная плотность информации?)
- Визуально сочетаются, но не идентичны
- Pro — более профессиональный, насыщенный информацией
- Lite — минималистичный, «вставил ссылку и подключился»

## Скоуп: Pro-версия

### Экраны (8 штук + кастомное окно)
1. **Установка (Setup)** — первый запуск, начальная настройка
2. **Панель управления (Control Panel)** — главный экран, управление VPN
3. **Настройка VPN (VPN Config)** — конфигурация подключения
4. **Дашборд мониторинга** — статистика, графики (под вопросом — нужен ли?)
5. **Маршрутизация (Routing)** — продвинутая маршрутизация трафика
6. **Логи (Logs)** — просмотр логов приложения и VPN
7. **Настройки (Settings)** — настройки приложения
8. **О программе (About)** — версия, лицензия, ссылки
9. **Кастомное окно** — title bar, drag, resize, minimize/maximize/close

### Дополнительный функционал
- Drag & drop конфигов
- Трей-меню с управлением VPN
- Deep links (tt:// протокол)
- i18n (русский/английский)

## Скоуп: Lite-версия
- Значительно меньше экранов
- Фокус: вставить ссылку → настроить маршрутизацию → подключиться
- Детально прорабатывается после завершения дизайн-системы Pro

## Скоуп: Установщик
- Кастомный установщик, соответствующий дизайн-системе приложения
- Реализуется **последним** — наследует готовую дизайн-систему
- Должен учитывать будущее: автообновление, деинсталляция, silent install

## Подход к реализации

Выбран **гибрид D + A** (из exploration-сессии):

1. Определить бренд-идентичность (стиль, настроение)
2. Зафиксировать design tokens (цвета, типографика, spacing) — light + dark
3. Создать Storybook-приложение для визуальной проверки компонентов
4. Спроектировать ОДИН ключевой экран (панель управления) — проверить токены
5. Извлечь компонентную библиотеку из этого экрана
6. Применить к следующим экранам (расширяя библиотеку)
7. Lite-версия наследует дизайн-систему
8. Установщик — последний

### Storybook
- Отдельное веб-приложение для визуального тестирования всех компонентов
- Каждый компонент изолирован, виден во всех состояниях
- Переключение light/dark тема
- Разработчик проверяет компонент в Storybook → утверждает → компонент идёт в приложение
- Исторически был зафиксирован как `gui-app/.storybook/`
- Актуальная реализация на 2026-04-20 живет в `gui-pro/.storybook/`

### Implementation note (2026-04-20)
- shipped desktop v3 сейчас живет в `gui-pro/`, а не в `gui-app/`
- старые ссылки на `gui-app` в ранних v3 заметках нужно читать как исторический слой
- текущий source of truth для design-system implementation: `gui-pro/src/shared/styles/tokens.css`
- текущий source of truth для shared UI: `gui-pro/src/shared/ui/`
- текущий source of truth для Storybook: `gui-pro/.storybook/`
